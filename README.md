# nbdpsy-skills

NBDpsy 心理科普内容创作的五个 Agent Skill，覆盖「话题 → 官网 SEO 长文 → 小红书图文笔记 → 配图 → 竖屏短视频」全产线，每级产物都有对抗审查闸门。支持 **Claude Code** 与 **Codex** 一键安装。

| Skill | 作用 |
|-------|------|
| **seo-artical-creator** | 把心理科普主题写成面向 SEO + GEO 的 pillar 长文（查证优先 → GEO 化结构 → 合规校验 → 生成即发布），产物入官网博客。 |
| **xiaohongshu-creator** | 承接上面的长文，拆成多篇可直接发小红书的图文笔记：每篇 ~300 字正文 + 一套 6–9 页「PPT 式」轮播插图（每页含页面文字 + 可喂给 Gemini/GPT 的中文绘图提示词），并生成带一键复制按钮、按内容命名的预览页（`{笔记目录名}-preview.html`）。 |
| **text-to-video** | 把长文 / 小红书笔记转成带中文字幕的竖屏短视频：即梦 Seedance 2.0 生成画面（走会员积分）+ 豆包 TTS 旁白 + 纯 ffmpeg 合成。可衔接 xiaohongshu-creator，给每篇笔记按「文生」或「图生」生成视频。**需另配** dreamina CLI 登录 + 豆包 TTS key（可选，见下）+ ffmpeg（首次跑 `text-to-video/scripts/check_env.py --install` 自检自装）。 |
| **content-reviewer** | 内容产线的对抗审查员：对长文 / 笔记 / 配图 / 视频四类产物先跑确定性检查脚本、再逐条「找茬」核对合规清单，产出 PASS/FAIL 审查报告。必须由独立子代理执行（生产者不自审），是发布前最后一道闸门。 |
| **content-pipeline** | 内容产线总导演：运营只给一个话题，说一句「做一期 XX 的全套内容」，就自动把长文创作→审查→拆笔记→审查→出图→审查→出视频→审查整条流水线串完，每级产物过审才进下一级。适合非专业兼职运营的傻瓜式入口；各生产 skill 也可单独使用。 |

> 这些 skill 为 NBDpsy（咨询师全员北大硕博的纯线上华人心理咨询工作室）定制，绑定其品牌话术、心理科普合规红线（YMYL）与数据库结构。**他人使用需按自己项目改造** SKILL.md 里的品牌 / 数据库 / 发布相关部分。

## 一键安装

**一条命令自动搞定**——装好 5 个 skill 到你的 AI 工具，同时自动装好全部依赖（ffmpeg、中文字幕字体、Python 包、即梦 dreamina CLI）、跑一遍凭据向导，不用你分步操作。

### Linux / macOS

```bash
git clone https://github.com/Buxiulei/nbdpsy-skills.git
cd nbdpsy-skills
./install.sh           # 默认 all，等价于不带参数直接跑 ./install.sh；装完自动装依赖 + 跑凭据向导
# ./install.sh claude  # 只装 Claude Code（~/.claude/skills/）
# ./install.sh codex   # 只装 Codex（~/.agents/skills/ + ~/.codex/skills/ 符号链接）
# ./install.sh agents  # 只装 Agent 标准目录（~/.agents/skills/）
```

远程一行安装（不想先 clone）：

```bash
curl -fsSL https://raw.githubusercontent.com/Buxiulei/nbdpsy-skills/master/install.sh | bash
```

### Windows

```powershell
git clone https://github.com/Buxiulei/nbdpsy-skills.git
cd nbdpsy-skills
.\install.ps1          # 默认 all；也可传 claude / codex / agents
```

远程一行安装：

```powershell
irm https://raw.githubusercontent.com/Buxiulei/nbdpsy-skills/master/install.ps1 | iex
```

> ⚠️ **Windows 下即梦 dreamina CLI 不会自动安装**（官方未验证 Windows，视频生成建议在 WSL 里装，向导会提示）；其余依赖照常自动装。

**只有两件事这条命令替不了，必须你自己动手**：

1. **即梦（视频生成）扫码登录**：终端跑 `dreamina login --headless`，用**抖音 App 扫码**（不装/不登录不影响其余四个 skill）。
2. **向管理员索要「凭据配置包」**：装完如果提示缺 Key（比如官网博客发布用的 Key），找管理员要，发给 AI 就能一键导入——见下方「凭据手册」。

装完（Claude Code 需重启一下）说「写一篇 XX 主题的 pillar 长文」「把这篇长文拆成小红书」或「做一期 XX 的全套内容」即会触发对应 skill。

> **Claude Code 插件市场**（另一种装法，只装 skill 本体，不带自动依赖安装）：
> ```text
> /plugin marketplace add Buxiulei/nbdpsy-skills
> /plugin install nbdpsy-content@nbdpsy-skills
> ```
> 用这种方式装完，记得手动补跑一遍 `python3 setup.py`（见下方「手动 / 排障」章节）把依赖和凭据配齐。
> 插件市场装完没跑 setup 也不怕：首次使用或遇到报错时，skill 会自检自愈，缺的 Python 包自动补装、缺系统件/凭据会明确告诉你怎么办。

## 手动 / 排障：单独跑环境向导

> 上面的 `install.sh` / `install.ps1` 装完会**自动跑一遍**下面这套向导，平时不需要你手动执行。以下场景才用得上：用 Claude Code 插件市场装的（不带自动依赖安装）、只想补装/重装某一项、或者想看看还缺什么。

```bash
python3 setup.py                # Linux/macOS
py setup.py                     # Windows
```

跑一遍：探测系统 → 装 ffmpeg / 中文字幕字体 → 装 Python 依赖 → 自动装 dreamina CLI（视频生成用，可选；装完还需你自己终端扫码登录，见下）→ **凭据向导** → 每个 skill 冒烟测试（`--help` 级别）→ 打印终检报告。幂等，可反复跑。参数：`--yes`（非交互：能装的都装，含 dreamina CLI 自动安装，凭据只报缺不问）、`--skip-credentials`（跳过凭据向导）。

**凭据向导会问 3 个 key（最省事的方式是找管理员要「凭据配置包」一键导入，见「凭据手册」；下表是手动逐项配置的备用说明）：**

| 变量 | 是否必需 | 用途 | 缺失怎么办 |
|------|---------|------|-----------|
| `NBDPSY_BLOG_API_KEY` | 必需 | seo-artical-creator 调用官网博客发布 API | 找管理员要「凭据配置包」（推荐）；或自己去管理后台 `manage.nbdpsy.com` → 博客 → API Keys 新建 |
| `VOLC_TTS_APPID` / `VOLC_TTS_ACCESS_TOKEN` | 可选 | text-to-video 豆包高音质旁白 | 找管理员要（凭据配置包会一并带上）；留空跳过也行，旁白改用免费引擎 `tts_gen.py --engine edge` |

答案存进**用户级凭据文件**（`~/.config/nbdpsy/secrets.env`，Windows 为 `%APPDATA%\nbdpsy\secrets.env`，可用 `NBDPSY_SECRETS` 环境变量覆盖路径）——**在任何仓库之外，绝不会被 git 跟踪或提交**；向导只报「已配置」，不会读取或回显已存在凭据的真实值。`NBDPSY_BLOG_API_KEY` 由三个生产 skill 通过 `shared/nbdpsy_common.py` 的 `get_secret()` 统一读取（优先级：环境变量 > 内容工作区 `.env` > 用户级凭据文件）。想跳过逐项问答，直接运行 `python3 <任一 skill>/scripts/nbdpsy_common.py secret import <凭据包文件>` 即可一键导入管理员给的整包凭据；`... doctor` 可随时自检还缺哪些。

> `VOLC_TTS_*` 的凭据探测按三级链执行（优先级从高到低）：
> 1. **环境变量** — 当前 shell 已设置的值
> 2. **skill 目录 `.env`** — `cp text-to-video/.env.example text-to-video/.env` 后手填 `VOLC_TTS_APPID` / `VOLC_TTS_ACCESS_TOKEN` / `VOLC_TTS_CLUSTER`
> 3. **用户级凭据文件** — `~/.config/nbdpsy/secrets.env`（setup.py 向导写入的值）
> 
> 未在前两级配置时，text-to-video 会自动回退到用户级凭据；若三级都未配置，旁白改用免费引擎 `tts_gen.py --engine edge`。

**dreamina CLI 本身向导会自动装好，但登录这一步它代劳不了**：终端跑 `dreamina login --headless`，用**抖音 App 扫码**，凭据存本地 `~/.dreamina_cli/`；不装/不登录也不影响其余四个 skill。

## 工作区说明

各 skill 的产物（草稿、笔记、视频工作目录）都落在**内容工作区**，路径按以下顺序解析（`python3 <任一 skill>/scripts/nbdpsy_common.py workspace` 可查询实际路径）：

1. `NBDPSY_WORKSPACE` 环境变量——设了就优先用它。
2. 未设时，若当前目录下存在 `seo-geo/content`（如在 NBDpsy 仓库根运行）——用它，向后兼容旧约定。
3. 都没有——落回 `~/nbdpsy-content`。

## 工作流（五个 skill 串起来）

```
心理科普话题
   │  content-pipeline（一句话触发：「做一期 XX 的全套内容 / 一条龙 / 全流程」）
   │
   ├─① seo-artical-creator
   │     → 官网博客 pillar 长文（4000+ 字，查证优先，生成即发布，走 publish API）
   │     └─ content-reviewer 审查（checklist-article）
   │        FAIL → 报告交回定向返工 → 复审（最多 2 轮，仍 FAIL 停手找人）
   │
   ├─② xiaohongshu-creator（自动拉取 ① 刚发布的 slug）
   │     → 5–8 篇小红书图文笔记（发布文案 + 配图提示词 + {笔记目录名}-preview.html）
   │     └─ content-reviewer 审查（checklist-note）→ 同返工协议
   │
   ├─③ 出图（宿主自适应，无独立 skill，属 xiaohongshu-creator 第 6 步）
   │     宿主有生图能力（如 Codex）  → 直接按提示词生成 images/post-NN/P01…PNN.png
   │     宿主无生图能力（如 Claude Code）→ 真停等：把 {笔记目录名}-preview.html 路径
   │                                      + 提示词发给运营，回传 images/post-NN/ 后才继续
   │     └─ content-reviewer 审查（checklist-images）→ FAIL 只重出问题页
   │
   ├─④ text-to-video（driver 十步 + 2.5 分镜确认页：解析笔记→精修分镜→
   │     生成 {视频目录名}-storyboard.html 分镜确认页（图生模式且宿主不会画图时
   │     真停等收图）→逐镜旁白→写回时长→提交生成→并行取片→BGM→拼合成清单→合成→…）
   │     → 带中文字幕的竖屏短视频 final.mp4
   │     └─ content-reviewer 审查（checklist-video）→ FAIL 只重跑问题镜
   │
   └─⑤ 人工上传（小红书图文 / 视频号·抖音·B站短视频，均无发布 API，最后一步必人工）
```

各 skill 也可单独使用，不必经过 content-pipeline；content-reviewer 必须由独立子代理执行，生产 agent 绝不自审。

## 📖 使用手册

> 写给运营看的。这里不教你跑命令——你只需要在 Claude Code / Codex 的对话框里打字，告诉 AI 你要什么，剩下的交给它。命令行内容只在"AI 会替你自动执行"或排障时用来对照，**一般无需手动执行**。

### 1. 三分钟上手（最常用的一句话）

在 Claude Code 或 Codex 的对话框里直接打字：

```
做一期「XXX」的全套内容
```

（把 XXX 换成你想做的心理科普话题，比如"社交焦虑"）说完这一句，AI 会自动依次：写长文并发布到官网 → 拆成小红书图文笔记 → 出配图 → 出竖屏短视频。整条流水线不用你再发第二句话。

**你只需要做的事**：

1. **给一个话题**——一句话，越具体越好（如"回避型依恋的人怎么谈恋爱"比"依恋"更好）。
2. **出图时搭把手**（仅当 AI 提示需要你帮忙时）：AI 会给你一个网页地址，让你打开、复制提示词喂给 Gemini/GPT 出图，图下载后放进指定文件夹，回复"图片好了"即可继续。如果你用的 AI 工具本身能直接画图（如 Codex），这一步会被自动跳过。
3. **最后人工发布**：小红书图文、视频号/抖音/B 站的视频，平台不开放自动发布接口，AI 做完会把文件路径和"怎么发"的要点交给你，由你手动上传。

**中途放心**：每做完一级产物（长文/笔记/配图/视频），AI 都会自动叫另一个"审查员"角色出来找茬——查错字、查断链、查有没有编造数据、查有没有违规词。审查不通过会**自动返工**，你不用管；只有连续两轮返工还是不过，AI 才会停下来找你。

### 2. 分场景用法

不想走全套流程，也可以单独只要某一段产出。以下每个场景给一句可以直接对 AI 说的话作参考（意思对即可，不用背诵原话）。

#### 场景 A · 只写一篇官网长文并发布

**跟 AI 说**：「写一篇『XXX』主题的 pillar 长文」

**会发生什么**：AI 选题、上网查证数据和专家引语、按官网 SEO/GEO 规范撰写 4000+ 字长文，自查合规和字数后，**直接调用官网博客 API 发布上线**（不是先出草稿等你确认——这套流程是"发布优先、事后复查"）。

**你要做什么**：发布后 AI 会给你文章链接，抽空点开看一眼内容和署名对不对。发现问题（内容有误、署名错了），去管理后台 `manage.nbdpsy.com` 直接下架（改成草稿或删除）或编辑，不用回来找 AI 重新走流程。

**产物在哪**：线上地址固定是 `https://www.nbdpsy.com/blog/<slug>`（slug 是 AI 起的英文短链接名，会告诉你）。

**耗时/花费**：主要耗时在 AI 查证与撰写，无需你等待操作；不消耗任何付费额度。

#### 场景 B · 把已发布的文章拆成小红书笔记

**跟 AI 说**：「把 slug 为 XXX 的文章拆成小红书」，或者不知道 slug 时说「把最新那篇拆成小红书」（AI 会列出最近的文章让你选）。

**会发生什么**：AI 自动从官网拉取这篇长文全文，拆成 5–8 篇独立的小红书笔记草稿，每篇配一套轮播配图的绘图提示词，并生成一个带"一键复制"按钮的预览网页。

**你要做什么**：这一步产出的是文字稿，还没有图，通常紧接着会问你要不要出图（见场景 C）。

**产物在哪**：每篇笔记一个 `post-01.md` … `post-0N.md` 文件，加一个按内容命名的预览页 `{笔记目录名}-preview.html`（AI 会给你具体路径和文件名）。

**耗时/花费**：几分钟内出稿；不消耗任何付费额度。

#### 场景 C · 出图（这一步最需要你配合，重点讲清楚）

这是唯一一步不同工具体验差很多的场景——取决于你用的 AI 本身能不能直接画图。

**如果你用的是 Claude Code（一般不能直接生成图片）**：AI 会**真正停下来等你**——不会假装自己已经出好图就往下走。默认把第 1 篇笔记（post-01）的全部页提示词直接贴在对话里；其余篇你说要哪篇它就贴哪篇——所有篇的提示词在预览页里都有一键复制按钮。你出图回传后它才会继续：

1. AI 会告诉你一个 `{笔记目录名}-preview.html` 预览页的具体路径和文件名，让你用浏览器打开它。
2. 页面里每一页配图都有一段「绘图提示词」和一个「复制」按钮。从第一页开始，点复制，把提示词粘贴给 Gemini 或 GPT，请它画一张图。
3. 下载生成的图片，放进 AI 告诉你的那个子文件夹（形如 `笔记目录/images/post-01/`，每篇笔记一个独立子文件夹）。
4. 按页码顺序命名成 **两位数**：第一页 `P01.png`，第二页 `P02.png`……以此类推。
5. 全部页面都放好图之后，回复 AI 一句「图片好了」，它会继续检查并进入下一步。

**如果你用的是 Codex（本身能画图）**：AI 会自己逐页生成图片放进对应文件夹，你不用做这步操作，只需要抽空打开图片文件夹扫一眼，确认没有错字、风格没有跑偏就行。

**产物在哪**：`笔记目录/images/post-NN/`，每个子文件夹里是该篇笔记的一套配图。

**耗时/花费**：人工出图这一步耗时取决于你操作 Gemini/GPT 的速度（一篇笔记 6–9 张图，一次拆分通常 5–8 篇），是全流程里人工投入最多的一环；出图工具本身的费用由你所用的 Gemini/GPT 账号承担，与本工具无关。

#### 场景 D · 把笔记做成视频

**跟 AI 说**：「把第 1 篇笔记做成视频」

**会发生什么**：AI 把这篇笔记的文字和配图转成分镜脚本，逐镜生成中文旁白、调用即梦（Seedance）生成画面，最后合成一条带中文字幕、按平台合规默认打上"AI 生成"角标的竖屏短视频。

**开跑前先给你一页"分镜确认页"**：不管走哪种画面模式，AI 正式提交生成前都会先做一个 `{视频目录名}-storyboard.html` 分镜确认页——每一镜的旁白脚本、字幕文字、绘图提示词（一键复制）都列在里面，方便你扫一眼台词对不对。

- 如果这条走的是默认的"文生"模式（画面靠文字提示词直接生成，不需要参考图），这一页只是给你看看，AI 会直接继续，不用你做什么。
- 如果走的是"图生"模式（用图片当画面首帧，风格更贴近你的小红书图文）且你用的 AI 本身不会画图（如 Claude Code）：AI 会**真正停下来等你**——把确认页路径和每一镜的绘图提示词都发给你，你按提示词生成图片，命名成 P01.png 两位数（写成 P1.png 也认）……放进 AI 告诉你的目录，回复"图片好了"它才会继续。用 Codex 之类本身能画图的工具时这一步会自动完成，不用你操心。

**你要提前准备**：即梦账号需要**提前在终端扫码登录过一次**（一般无需手动执行，AI 需要时会提示你跑 `dreamina login --headless`，用抖音 App 扫码，这一步 AI 代替不了）。

**排队心理准备**：即梦画面生成要排队，正常情况下可能要**几十分钟到几小时**；如果你很急，告诉 AI「这条急用」，它会走加速通道（多花积分，实测能把等待压到几分钟量级）。

**产物在哪**：`videos/<篇目>-NN/final.mp4`。

**你要做什么**：成片做完并通过 AI 的自动审查后，**发布前自己再完整看一遍**（音画是否同步、字幕有没有错别字、有没有奇怪的画面），确认无误再上传到视频号/抖音/B站。

**耗时/花费**：全流程里**最耗时也最烧钱**的一步——排队是最大变量；生成画面消耗你即梦账号的会员积分，只有成功生成才扣分，越急（走加速通道）花的积分越多。

#### 场景 E · 单独审查一份产物

**跟 AI 说**：「审查这篇长文」/「检查这批图」/「验收这条视频」

**会发生什么**：AI 会另开一个独立的"审查员"角色（不是刚才写内容的那个），先跑一遍确定性检查脚本（字数、死链、违禁词、图片尺寸等），再逐条人工核对（编造数据没有、图内有没有错字、音画是否同步等），最后出一份 PASS/FAIL 的审查报告。

**你要做什么**：等报告即可；FAIL 时报告里会写清楚哪里有问题、怎么改。

**产物在哪**：`review-report.md`，跟被审的产物放在同一个目录里。

**耗时/花费**：几分钟；不消耗付费额度。

### 3. 凭据手册

**最省事的方式（推荐）**：找管理员要一份「凭据配置包」——管理员在管理后台「博客 → API Keys → 生成凭据配置包」点一下就能生成，把生成的整段发给你。你原样发给 AI，或者自己运行 `python3 <任一 skill>/scripts/nbdpsy_common.py secret import <凭据包文件>`，都能一键导入全部凭据，不用碰下面表格里的任何一项。想自己确认还缺哪些，跑一下 `python3 <任一 skill>/scripts/nbdpsy_common.py doctor` 就会列出来。下面的表格是"手动逐项配置"的备用说明。

以下这些"凭据"（即密钥/账号）AI 会在需要时主动向你要，你只管照着"怎么填"那一栏做就行，**不需要自己去改任何配置文件**。

| 凭据名 | 干什么用 | 必需? | 去哪获取 | 怎么填 |
|---|---|---|---|---|
| `NBDPSY_BLOG_API_KEY` | 授权 AI 把长文发布到官网博客（场景 A 用） | **必需** | 找管理员要「凭据配置包」（推荐）；或登录管理后台 `manage.nbdpsy.com` → 博客 → API Keys → 新建 | 首次发布时如果这个 key 还没配，AI 会提示你，你把在后台新建出来的那串值粘贴给它即可；AI 会自动存进你电脑本地的凭据文件，不用你手动改任何文件 |
| `VOLC_TTS_APPID` + `VOLC_TTS_ACCESS_TOKEN` | 让视频旁白用火山引擎「豆包」大模型配音，音质更自然（场景 D 用） | 可选 | 火山引擎控制台 → 语音技术 → 语音合成大模型，开通音色后拿 APPID / Access Token | 同上，告诉 AI 这两个值即可自动存好；**不申请也没关系**，AI 会自动改用免费的 edge 引擎配音，只是听感稍逊一筹 |
| 即梦（dreamina）登录 | 让 AI 能调用即梦 Seedance 生成视频画面，消耗你的即梦会员积分（场景 D 用） | 仅"出视频"时需要 | 用你已有的抖音 App 账号，不需要单独申请 key | 这一步 AI 帮不了忙：需要你自己在终端跑一条命令 `dreamina login --headless`（一般无需手动执行，AI 需要时会提示你），用抖音 App 扫码登录一次；登录状态只存在你这台电脑本地 |

**存放位置与安全性**：以上凭据只会存进你电脑本地的用户级文件（Linux/macOS 是 `~/.config/nbdpsy/secrets.env`，Windows 是 `%APPDATA%\nbdpsy\secrets.env`），这个文件**在所有代码仓库之外，绝不会被提交进 Git**。AI 只会告诉你某项"已配置"，**永远不会把你填过的真实值念出来或回显给你或任何人**。

### 4. 常见问题 FAQ

**① 发布测试草稿怎么删？**
AI 发布时如果用了"草稿"模式（一般是你明确要求"先别发布"时），这个草稿会**真实写进线上数据库**，跟正式文章存在同一张表里，系统没有自动清理功能。测试完想删掉，去管理后台找到这篇文章手动删除或下架。只想看看 AI 打算发的内容长什么样、完全不想落库，让 AI 只打印内容不真发请求即可。

**② "slug 已存在被跳过"是什么意思？**
这是一层保护：同一篇文章的网址短名（slug）如果线上已经有了，AI 不会覆盖它——因为线上那篇可能已经被管理员改过。AI 会自动跳过、告诉你"skipped"。如果你确实想拿新内容覆盖线上那篇，明确告诉 AI「更新这篇文章」而不是"发布"，它会切换到更新模式，且只改你实际提供的部分，其余保持线上原样。

**③ 视频排队很久正常吗？**
正常，这是即梦平台的真实排队现象，跟 AI 或本工具无关。高峰期排队可能要几十分钟到数小时；如果你告诉 AI「这条急用」，它会走加速通道，实测能把等待压缩到几分钟，代价是多花一些积分。

**④ Windows 能出视频吗？**
即梦（dreamina）官方目前没有正式验证过 Windows，直接在 Windows 上装可能不稳定，建议在 WSL（Windows 自带的 Linux 子系统）里跑视频生成这一步。写长文和拆小红书这两个场景在 Windows 上没有这个限制。

**⑤ 跳过豆包配音会怎样？**
不影响整体流程能不能跑通，AI 会自动切换成免费的 edge 引擎配音，能正常出片，只是听感上不如豆包自然、音色选择也更少。想升级随时可以按凭据手册补上火山引擎的 key。

**⑥ 新文章为什么官网列表页暂时看不到？**
文章详情页（点进去看全文的那个页面）发布后**几乎立即可见**；博客首页的列表页有 5 分钟左右的自动刷新缓存，最多等 5 分钟左右刷新一下页面，新文章就会出现在列表里，不用等一整天。

**⑦ 审查两轮都 FAIL 怎么办？**
这是设计好的安全阀：同一份产物最多允许自动返工复审两轮。如果第三轮还是不过，AI 会主动停下来，不再自己反复重试，并把三轮各自的问题写进审查报告（哪些改好了、哪些反复出、哪些是新冒出来的）。这时候按报告里的建议自己处理，或者找懂这套流程的人帮忙看一眼即可。

**⑧ 图放错文件夹/命名错了怎么办？**
每篇笔记的配图必须放进对应的子文件夹、按两位数页码命名（比如第 1 篇笔记的第 1 张图是 `images/post-01/P01.png`，第 9 页是 `P09.png`）。放错文件夹或命名不对，AI 后续检查图片、或者拿图做视频时会找不到对应的文件。发现放错了，直接把文件移到正确的子文件夹、重新按规则命名，再回复 AI 一声即可，不需要从头再走一遍前面的步骤。

**⑨ 换新电脑怎么迁移？**
在新电脑上重新跑一遍 `install.sh`（Windows 是 `install.ps1`）即可，依赖和凭据向导都会自动跑完；凭据这块最省事是再找管理员要一次「凭据配置包」导入。即梦（视频生成）账号需要在新电脑上重新扫码登录一次——它和文章/小红书用的凭据是分开存的，不会跟着搬家。

### 6. 排障速查

| 症状 | 原因 | 怎么办 |
|---|---|---|
| 装完之后跑环境向导，某一项显示 ✗（缺依赖） | 对应工具没装，或系统没给自动安装的权限 | 向导的终检报告里会直接给出对应的手动安装命令，照着在终端跑一遍（一般无需你自己想命令，AI 会帮你贴出来），再重跑一次向导确认变 ✓ |
| 发布长文时提示 `MISSING:NBDPSY_BLOG_API_KEY` | 缺官网博客发布用的 API Key | 按「凭据手册」，找管理员要「凭据配置包」发给 AI 一键导入（或自己去 `manage.nbdpsy.com` 建一个新 Key 发给 AI），存好后会自动重试 |
| 发布长文失败，报错像网络问题（超时/连接不上） | 网络暂时不通，或后端服务暂时打不开 | 让 AI 稍后重试即可；发布带幂等保护，同一篇不会被重复发布两次，可以放心重试 |
| 生成视频时报错找不到中文字幕字体 | 电脑没装中文字体（Noto Sans CJK 之类） | 按报错提示装字体（一般是一条 `apt-get`/`brew` 命令，AI 会帮你贴出来）；如果你电脑上已经有别的中文字体文件，告诉 AI 那个字体文件的路径也能顶上 |
| 生成视频卡在"即梦未登录/积分读取失败" | 即梦（dreamina）账号没登录或登录过期 | 在终端跑 `dreamina login --headless`（一般无需手动执行，AI 需要时会提醒你），用抖音 App 扫码重新登录 |
| AI 说某项审查脚本"跑出来 exit 不是 0"，让你看一段 JSON | 这是确定性检查脚本发现了硬性问题（比如字数不够、链接打不开、命中了违禁词） | 不需要自己读 JSON，直接让 AI 把这段结果"翻译成人话"——它看得懂里面的字段，会直接告诉你具体差在哪、需不需要它去改 |

## 合规红线（五个 skill 共用）

- 不编造数字 / 出处 / 专家原话（YMYL 健康领域）。
- 不用「治疗 / 诊断 / 治愈 / 医院 / 医生」自我描述；不用极限词；不夸大。
- 小红书侧额外：任何位置不放站外导流（微信 / 二维码 / 外链）。
- 视频侧额外：图内不出现文字（CJK 渲染必乱码）；driver 产线默认打「AI 生成」角标。
- 危机声明（全国统一心理援助热线 12356）每篇 / 每条在位。
- 生成产线只在开发机/专用机跑，**绝不部署到生产服务器**。
