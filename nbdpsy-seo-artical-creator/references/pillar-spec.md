# Pillar 长文硬性规格（GEO 化模板 · 所有博客长文共用）

> 依据：Princeton GEO 研究（带出处统计 +41%、专家引语 +28% 被引概率）、Bing 官方诊断（内容不足/描述太短判低质）、YMYL E-E-A-T 要求。下列每条"硬性要求"违反任一即不合格。
>
> 约定：`{SKILL_DIR}` 指本 skill 根目录（本文件上一级目录）。

## 硬性要求清单（R1 复审逐条对照）

1. **篇幅**：正文 3000–5000 字（不含 FAQ 与参考文献）。口径=**纯汉字数**：`grep -oP '[\x{4e00}-\x{9fa5}]'` 计数，英文/数字/标点不算。用 `python3 {SKILL_DIR}/scripts/count_hanzi.py` 验（区间外——不足或超出——都 exit 2 判不合格）。首批四篇实测 4379–4862 字，3000 是底线不是目标。
2. **答案前置**：H1 之后第一段就是 TL;DR 直接结论（80–120 字，能脱离上下文独立回答标题问题）。正文前 1/3 是 AI 引擎引用热区，结论埋后面=放弃被引。
3. **带出处统计块**：全文 ≥3 处真实统计数据，每处紧跟来源（机构 + 年份 + 可点链接）。数字必须经网络搜索并打开网页核实存在且口径正确，**严禁编造数字或 DOI/PMID**。
4. **专家引语**：≥2 处已发表文献/公开演讲中**真实专家**的原话或紧密转述（注明出处）。文章默认署名胡佰亿（真人），**绝不得把虚构引语安到他或任何 NBDpsy 咨询师名下**。
5. **FAQ 段**：文末 5–8 个 Q&A，每个 Q 用用户真实搜索短语，A 第一句直答。frontmatter 的 `faq` 与正文 FAQ 段保持一致（页面 FaqBlock 组件按 frontmatter 渲染 FAQPage schema）。
6. **参考文献**：≥6 条，真实可点（DSM-5-TR/ICD-11 官方页、PubMed/DOI、权威机构指南、领域奠基著作）。逐条打开网页核实 URL 可达后才写入。
7. **敏感词红线**：不得用「治疗/诊断/治愈/医院/医生」描述本工作室服务（学术名词如 PTSD、EMDR、CBT、对心理治疗研究文献的转述可保留）；不得夸大效果（「彻底摆脱」「100%」「根治」禁用）；合规口径=「咨询/干预/评估/陪伴」。
8. **危机声明**：文末固定加——「本文不构成医疗建议；如处于心理危机请拨打希望24热线 4001619995 或全国统一心理援助热线 12356」。
9. **内链**：正文自然嵌入 2–4 处站内链接（`/services/*`、`/counselors`、相邻 pillar 的 `/blog/{slug}`），锚文本用精确关键词，禁「点击这里」。四篇 pillar 彼此交叉内链，织成主题网。
10. **结构**：H2 用用户搜索短语作小标题；至少一张对比表格（markdown table）；段落 ≤150 字（手机竖屏可读）。

## 文内引用规范（强制）

正文每处统计数据、专家引语、事实断言之后，须紧跟数字标注 `[[n]](来源URL)`（渲染为可点击的 `[n]`）：n 对应文末「## 参考文献」有序列表的序号；每条参考文献须至少被正文标注一次，不留孤儿引用；标注里的 URL 须与该条参考文献的 URL 一致。

正例：`CPTSD 患者中约 92% 报告长期人际关系困难 [[3]](https://doi.org/xxxx)。`

背景：此前三篇文章各用一套自创样式（一篇正文甚至零链接），审查无从核对"数字↔出处"对应关系。统一为数字标注后可机器校验。用 `python3 {SKILL_DIR}/scripts/lint_markdown.py <file.md> --citations <参考文献条数>` 校验（citation-marker 规则；有未覆盖序号即不合格）。

## 中文加粗排版规范（强制，渲染兼容）

背景（生产事故根因）：CommonMark 强调（`**...**`）判定依赖"侧翼"规则——闭合 `**` 右侧若紧跟文字（含中文），但左侧是全角标点（如 `……。**很多人`），右侧翼不成立；开启 `**` 左侧紧贴文字、右侧是全角开引号/开括号（如 `字**「话」`），左侧翼同样不成立。侧翼不成立时渲染器（本站用 react-markdown）不解析为加粗，原样输出裸 `**`——线上文章曾因此 10 处加粗渲染失败。

硬性要求：
- 加粗内容**不得以全角标点结尾**（。！？；：、，等）——标点放在 `**` 外。
- `**` 不得紧贴在文字与左引号/左括号（“『「（《）之间——引号/括号放在 `**` 外。

正例：`**核心结论**：CPTSD 与 PTSD 的关键区别在于……`　`**要点**。`
反例（渲染为裸星号，即生产事故复现）：`……**要点。**紧跟正文`　`他说**「可以」**`

用 `python3 {SKILL_DIR}/scripts/lint_markdown.py <file.md>` 校验（bold-flanking 规则，围栏代码块内跳过）。

## 交付格式

写入**内容工作区 drafts 目录**下的 `{slug}.md`（工作区实际路径用 `python3 {SKILL_DIR}/scripts/nbdpsy_common.py workspace` 查询；在 NBDpsy 仓库根运行时自动解析为 `seo-geo/content`，向后兼容），YAML frontmatter + markdown 正文：

```yaml
---
title: （H1，≤30 字，含核心关键词）
slug: （拼音 ASCII，连字符分隔）
excerpt: （≤150 字摘要）
category_slug: （按主题从下方「固定分类清单」选一个，勿照抄示例默认值）
tags: [标签1, 标签2]   # 3–6 个，自由词（后端自动 upsert），如 情绪内耗/依恋/自我关怀
meta_description: （120–160 字，含长尾词 + 直接回答句）
faq:
  - q: ...
    a: ...
citations:
  - title: ...
    url: ...
    source: （机构/期刊 + 年份）
internal_links:
  - keyword: ...
    url: ...
target_keywords: [主词, 长尾词...]
author_name: 胡佰亿            # 默认署名（真人），脚本据此署名；如需别的署名改这里
---
（正文 markdown，首行即 `# {title}`；发布脚本会剥掉首行 H1 避免与页面 hero 重复）
```

## API 字段映射（`publish_post.py` 构建 payload）

发布走 `POST /api/external/blog/posts`（更新用 `--update` 改走 `PUT .../posts/{slug}`）。**值为空的键不发送**（键缺失或值为 null 都不进 payload），避免 null 覆盖后端已有值——`--update` 时未发送的字段保持后端原值不变。

| frontmatter 键 | API 字段 | 说明 |
|---------------|----------|------|
| `title` | `title` | 必填（缺失后端 400） |
| `slug` | `slug` | 缺省用文件名（去 `.md`）兜底；冲突返回 409 → 脚本记 skipped 不覆盖 |
| `excerpt` | `excerpt` | |
| （正文） | `content_markdown` | 剥掉首行 H1 后的正文，必填 |
| `category_slug` | `category_slug` | 后端换算 category_id；slug 不存在返回 400，不静默忽略 |

**固定分类清单（category_slug 只能从这里选，绝不自造——服务端 400）**：

| slug | 分类 | 收什么 |
|---|---|---|
| `trauma-healing` | 创伤与疗愈 | CPTSD/创伤反应/闪回/内在批判者/疗愈路径 |
| `emotion-self` | 情绪与自我 | 情绪内耗/焦虑抑郁科普/自我批评与关怀/自我价值/习惯与调节 |
| `relationships` | 亲密与家庭 | 依恋/亲密关系/婚姻/家庭/追逃循环 |
| `workplace` | 职场心理 | 职场倦怠/效率/工作中的心理健康 |
| `overseas-students` | 留学生心理 | 留学生与海外华人的适应/情绪支持/中文咨询（业务主线，优先产出） |
| `psych-101` | 心理科普 | 通用/趣味科普（消费心理/睡眠/MBTI 等），以上都不贴切时的兜底 |

**判据**：看文章的核心受众场景归属，一篇只选一个；两类都沾时选更具体的那类（如「CPTSD 的亲密关系」→ trauma-healing）。要新增分类须先让管理员在生产库 `blog_categories` 建好（INSERT name/slug/description/sort_order）再使用，勿先用后建。

**标签（tags）选型三规则**（分类管导航骨架，标签管细粒度聚合与内链——两层各司其职）：

1. **优先复用已有标签**：写 tags 前先 `curl -s https://database.nbdpsy.com/api/public/blog/tags` 看现有标签库，同义概念**必须用库里的原词**（已有「自我关怀」就别再造「自我照顾」）——tags 是自由词自动入库，同义分裂会让标签聚合页与相关文章推荐失效。库里没有的新概念才新造。
2. **命名规范**：2–6 字中文名词短语（专业术语如 CPTSD/ADHD 可用英文缩写），主题词＋人群词＋方法词搭配（如 `[CPTSD, 情绪闪回, 自我关怀]`）；不用句子、不带 `#`（那是小红书 hashtags 的写法）、不与所选分类名重复（分类已承载的词不必再打成标签）。
3. **对齐搜索意图**：3–6 个里至少 1 个来自本篇 `target_keywords` 的核心长尾词——标签页是长尾词的聚合着陆面，与 SEO 主线互相喂。
| `tags` | `tag_names` | 必须是数组（否则丢弃并告警）；后端 upsert 标签并关联——与旧 SQL 脚本"tags 不入库"不同，现在会发 |
| `author_name` | `author_name` | frontmatter > `--author` > 胡佰亿，恒发送 |
| （`--draft` 开关） | `status` | 默认 `published`（后端自动写 published_at=NOW()）；`--draft` → `draft` |
| `cover_image_url` | `cover_image_url` | 可选 |
| `video_url` | `video_url` | 可选 |
| `citations` | `citations` | `[{title,url,source}]` 数组，入 JSONB |
| `faq` | `faq` | `[{q,a}]` 数组，逐项过滤（q、a 均非空才发），入 JSONB |
| `meta_title` / `meta_description` | 同名字段 | 值非空才发送（SEO 元信息随文入库） |
| `target_keywords` / `internal_links` | （不发送） | 仅指导撰写，不入库 |

另：后端对 API 通道创建的文章自动记 `source_type='api'`（旧 SQL 脚本为 `'original'`）。如需先压草稿，加 `--draft`。

## 语气

专业但不冷漠；面向受困扰的成年人写作，**共情先于科普**（先接住情绪再讲知识，这是心理科普区别于普通 SEO 文的关键）；中文为主，专业术语首次出现给英文对照（如「复杂性创伤（Complex PTSD，CPTSD）」）。

## 查证流程的两个已知陷阱

- **NCBI 限速**：PubMed/PMC 对机器访问限速，打开网页核实时常拿不到全文。因为生成即发布、数字会带着胡佰亿署名上线，引文数字拿不准时**宁可不写也不硬猜**；若确需保留，frontmatter 标「待人工核对 PMID xxx」并在提醒管理员时点名复核。
- **标题像 ≠ 口径对**：打开网页后要确认正文真的支持你的论断，不能只看标题/摘要像就引。

## 字数返工闭环

首批四篇都是初稿不足 3000 → 按缺口补充真实内容（多一个分论点、多一张表、多一段共情案例）→ 重新计数，循环到达标。**靠真内容补，不靠注水/重复/废话凑数**——AI 引擎对低信息密度段落会降权。
