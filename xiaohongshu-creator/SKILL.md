# 小红书图文笔记创作（从 pillar 长文拆分）

承接 `seo-artical-creator` 产出的心理科普 pillar 长文，把一篇 4000+ 字的长文拆成 **5–8 条可直接发小红书的图文笔记**。每条笔记 = 一段 **~300 字正文** + 一套 **6–9 页"PPT 式"轮播插图**（每页给出"页面文字"+"中文绘图提示词"，提示词把要显示在图上的中文直接写进去，复制即可喂给 Gemini/GPT 出图）。

**项目背景一句话**：NBDpsy 是咨询师全员北大硕博的纯线上华人心理咨询工作室。官网 pillar 长文负责被搜索引擎/AI 引擎收录；小红书负责触达正在 emo、正在内耗、正在搜"怎么办"的真人。长文重信源与循证，小红书重共情与钩子——本 skill 的使命是把前者的可信内容，翻译成后者的语言，**绝不在翻译中编造新事实**。

## 三条贯穿全程的铁律（理解动机，别只背规则）

1. **不新增未验证事实**。统计数字、专家引语、出处只能沿用 pillar 原文里已核实过的内容。短文里出现的每个数字，都要能在源长文里找到。小红书没有参考文献区，但"编造一个数据被扒出来"对一个心理工作室是致命的——所以宁可少写一个数字，不可造一个。
2. **合规红线高于传播效果**。心理/健康类是小红书重点审核领域：极限词（最/第一/根治/100%/彻底）违广告法、医疗词（治疗/治愈/诊断）非医疗机构禁用、站外导流（微信/二维码/加群）直接限流封号。详见 `references/xiaohongshu-spec.md`，发布前用脚本扫一遍。
3. **小红书无 API，本 skill 只产出草稿**。配图按「出图（宿主自适应）」章节产出（宿主自动生成，或人工把每页提示词喂给 Gemini/GPT），但**最终发布永远是人工**：把正文+标签复制进小红书、配图上传。所以交付时要把"怎么发"也讲清楚。

多步任务先维护一份任务清单，每步勾掉。

> **路径约定**：以下 `{SKILL_DIR}` 指本 SKILL.md 所在目录；`{workspace}` 指内容工作区（实际路径用 `python3 {SKILL_DIR}/scripts/nbdpsy_common.py workspace` 查询）；`{note_dir}` 指一次拆分的产物目录 `{workspace}/xiaohongshu/{slug}/`。

## 完整流程（0–7 步，每步都有验证闸门）

```
0. 定位源长文               → 验证：拿到源长文的本地 md 路径（fetch_post.py 输出的 path）
1. 规划拆分                 → 验证：5–8 篇，每篇一个独立钩子、彼此不重叠、合起来覆盖原文核心
2. 写正文（每篇 ~300 字）    → 验证：count_xhs.py 字数达标 + 钩子/分点/CTA 齐 + 不新增未验证事实
3. 设计配图轮播（6–9 页）    → 验证：每页复用固定品牌基底 + 图中文字短 + 首页是钩子封面 + 视频参考图提示词页对页齐
4. 合规校验                 → 验证：check_compliance.py 逐篇全绿 + 危机声明每篇在位 + 语义自检过
5. 落盘 + 预览页 + 笔记审查  → 验证：一篇一文件写入 {note_dir}，含 00-overview 与 preview.html；笔记对抗审查 PASS
6. 出图（宿主自适应）+ 图审  → 验证：{note_dir}/images/post-NN/P01…PNN.png 齐全；图片对抗审查 PASS
7. 交付提醒（人工发布）      → 验证：给出文件路径 + 发布要点（首图最重要/3:4/标签/危机声明保留）
```

### 第 0 步 · 定位源长文

按输入类型三选一：

- **用户给了 slug** → `python3 {SKILL_DIR}/scripts/fetch_post.py --slug <slug>`：自动从官网公开 API 拉取全文，落到 `{workspace}/drafts/{slug}.md`（stdout 输出 JSON，`path` 字段即文件路径；本地已有同名文件直接复用缓存、不打网络。要落到别处加 `--out <目录>`）。
- **用户没给** → `python3 {SKILL_DIR}/scripts/fetch_post.py --list 10` 列最新文章（JSON 数组：slug/标题/发布时间）让用户选；`--list` 不带数字默认 10。全自动流水线中直接用上一级刚发布的 slug，无需询问。
- **用户直接给了本地文件路径** → 直接用。

拿到文件后先把全文读完——尤其是 TL;DR、各 H2、对比表、统计块、FAQ，这些是拆分的原料。

### 第 1 步 · 规划拆分

**规划 5–8 篇短文**：按 H2 主题**智能聚合**，不是机械每个 H2 一篇。判据：每篇围绕**一个能独立成立的钩子角度**（如「CPTSD 和 PTSD 到底差在哪」「童年情感忽视：那道看不见的伤」「复杂性创伤的 5 个自救步骤」）。太薄的 H2 合并、太厚的拆开。爆款角度参考 `references/xiaohongshu-spec.md` 的"选题口吻"。

把拆分大纲写进 `00-overview.md`（每篇：序号 / 钩子标题 / 对应原文哪些 H2 / 核心 takeaway / 拟用统计或引语）。

**验证**：每篇有独立钩子、彼此不重叠、合起来覆盖原文核心论点；篇数落在 5–8（原文确实太短才可少于 5，并说明理由）。

### 第 2 步 · 写发布文案（正文 ~300 字 + 可见标签）

每篇 post 的第一个内容块就是**复制即发的"发布文案"单元**——这是用户真正粘进小红书输入框的东西。块标题用 `## 发布文案（复制这一段直接发布）`，里面依次是：① 正文主体 ② 危机声明一行 ③ **可见的 `#标签` 行**。

> **标签必须可见**：标签除了写进 frontmatter 的 `hashtags`（供机器读取），**必须同时作为可见文本 `#标签1 #标签2 …` 附在文案末尾**。frontmatter 在网页/编辑器渲染时不显示，只放 frontmatter 等于「看不到标签、也没法复制」。小红书惯例标签贴在正文末尾。

打开 `references/xiaohongshu-spec.md` 对照写。骨架记忆点：

- **首句 3 秒钩子**：用痛点/场景/数字开场（"总觉得自己和别人不一样？"），抓不住人就划走了。
- **总-分-总**：分点用 `·` 或编号，每段 2–4 行、配 1 个克制的 emoji（治愈系不要满屏 emoji）。
- **口吻**：第一人称共情 + 干货清单 + 自救指南，口语化，像朋友说话。**共情先于科普**（这是心理内容区别于普通种草文的关键）。
- **结尾 CTA**：提问引评论 / "收藏不迷路" / "评论区抱抱"。
- **标题 ≤20 字**：数字体 / 痛点 / 悬念 / 身份标签四类钩子任选。
- **标签 5–10 个**：关联（精准品类如 #心理科普 #CPTSD）+ 热度（蹭当下）+ 长尾（细分/身份/地域，如 #留学生心理 #情绪内耗）。同时写进 frontmatter `hashtags` 与文案末尾可见标签行。

**字数默认 300 汉字**（用户指定值，只数正文主体、不含标签）。提醒：小红书爆款正文常在 500–800 字、信息更厚更易被收藏；如想提收藏率可把默认调到 500–600，但默认遵守 300（注意：count_xhs.py 的判定区间固定按 300 基准 210–450，调高目标字数时以人工核对为准）。

**验证**：`python3 {SKILL_DIR}/scripts/count_xhs.py <post.md>` 输出 JSON——`body_chars` 只数「发布文案」区块的汉字（剔除标签行），区间 210–450；`pages` 同时核轮播页数 6–9；以 `ok` 为 true 为过。此外：发布文案块含可见标签行；第 1 步规划的钩子有落地；正文里的数字都能在源长文找到出处。

### 第 3 步 · 设计配图轮播（每篇 6–9 页）

每条笔记配一套"像 PPT 一样"的轮播图。**核心规则见 `references/illustration-spec.md`**，记忆点：

- **每页 = 一张"知识卡"**：① 页面文字 ② 中文绘图提示词。**封面/末页精简**（大标题 + 一句）；**内容页信息要足**——大标题(≤14 字) + **3–5 条「加粗小标题(≤10 字) + 一行说明(≤24 字)」**，让读者刷到能扫到"对得上自己"的那条、停下来细看，而不是只看见一句空泛标题就划走。
- **图中文字直接写进提示词**：你用的 Gemini/GPT 能渲染中文，所以把这页要显示的中文用引号写进提示词，并指定位置与层级。整页信息多、但每个文字块（标题/小标题/说明）各自要短、独立成行——这样渲染更准、也更好扫。
- **首页 = 钩子封面**（最重要，决定点击率，不堆要点留悬念）；中间页 = 信息卡（一页讲透一个主题的 3–5 个点）；末页 = 行动建议 + 品牌一句话 + 危机声明。
- **全套复用固定品牌基底**：每页提示词都以同一段"风格基底"开头（莫兰迪低饱和治愈系插画 + 固定主色 + 固定人物卡），保证 6–9 张风格一致、整个号调性统一。
- **每篇笔记末尾追加 `## 视频参考图提示词` 节**（页对页的"去文字版"提示词）：每页一条 = 该页绘图提示词**剔除所有「图内文字」指令**，保留品牌基底与人物卡，末尾追加「画面中无任何文字」。用途：text-to-video 图生模式拿这套提示词出"无文字同款图"当每镜首帧——视频会把旁白烧录成字幕，图内若再有文字会和字幕打架。本 skill 只产这节提示词，不在本级出这套图（由 text-to-video 产线按需生成）。**节内每页用加粗标记 `**P1**`…`**PN**` + 围栏提示词，绝不用 `### PN` 标题**——`### PN` 是配图轮播的页数计数契约，重复使用会让 count_xhs.py 把页数翻倍误判。完整规格与模板见 `references/illustration-spec.md` 第 6 节。

**验证**：每页提示词都含品牌基底；图中文字够短；首页是钩子封面而非内页；页数落在 6–9；视频参考图提示词与轮播页一一对应且不含任何图内文字指令。

### 第 4 步 · 合规校验（发布前必须全绿）

```bash
# 高置信违禁词扫描（极限词/医疗违禁/站外导流）+ 危机声明在位检查
# 脚本一次收一个文件，逐篇跑；每篇 post-*.md 都要 ok=true（exit 0）
python3 {SKILL_DIR}/scripts/check_compliance.py {note_dir}/post-01.md
```

脚本测不出的**语义自检**（见 `references/xiaohongshu-spec.md` 的禁用词替换表）：

- 禁用「治疗/诊断/治愈/医院/医生」做自我描述或承诺 → 改「陪伴/梳理/觉察/自我调节/缓解」。
- 禁极限词「最/第一/根治/100%/彻底/绝对」。
- 禁站外导流：正文、评论、简介、图中都不能出现微信号/二维码/加群/外链（小红书会限流封号）。
- **危机声明每篇在位**（放正文末或末页）：`本文为心理科普，不构成诊断或治疗；如持续困扰请寻求专业帮助，全国统一心理援助热线 12356`。

### 第 5 步 · 落盘 + 预览页 + 笔记对抗审查

写入 `{workspace}/xiaohongshu/{pillar-slug}/`（即 `{note_dir}`）：

```
00-overview.md     拆分大纲（第 1 步产物）
post-01.md         frontmatter + ## 发布文案（正文 + 危机声明 + 可见标签行）+ ## 配图轮播 + ## 视频参考图提示词
post-02.md         ...
post-0N.md
preview.html       预览页（下面生成）
images/post-NN/    出图产物，每篇一子目录（第 6 步生成，P01.png…PNN.png）
```

一篇一文件，方便人工逐条复制去发。单篇文件格式见 `assets/example-xhs-note.md`（黄金范例，对照学习；范例暂未含「视频参考图提示词」节，该节规格以 `references/illustration-spec.md` 第 6 节为准）。

**生成预览页**——把拆分产物渲染成一个自带一键复制按钮的网页，区分"给人读"与"复制给 AI"两类内容：

```bash
python3 {SKILL_DIR}/scripts/render_preview.py {note_dir}   # 默认输出 {note_dir}/preview.html
```

- **发布文案/页面文字** → 暖纸阅读 UI，标签做成 pill，配「复制文案」按钮（整段粘进小红书）。
- **绘图提示词** → 深色等宽面板「保持原样」，每段右上角「复制」按钮（粘给 Gemini/GPT）。

**笔记对抗审查**：触发 content-reviewer（独立实例，绝不自审）按 `checklist-note.md` 逐篇审查；FAIL → 按报告定向返工后复审；PASS 才进入出图。

### 第 6 步 · 出图（宿主自适应）

先探测：你的宿主是否具备图像生成能力（能直接根据文字提示词产出图片文件）？

**有（如 Codex 等）**：按每页「绘图提示词」逐页生成 **1080×1440（3:4）** 图片，保存为 `{note_dir}/images/post-NN/P01.png … PNN.png`（**每篇一子目录**，`post-NN` 与笔记文件名对应——post-01.md 的图放 `images/post-01/`；页序号固定**两位数**）。生成后逐张自查图内中文有无错字，错字页用同一提示词重生成。完成后进入图片对抗审查。

**没有（如 Claude Code 等）**：输出 preview.html 的**绝对路径**，并把下面这段话告诉运营（`{note_dir}` 等占位符替换成真实路径，别让运营看到花括号）：

> 请打开这个页面：`{note_dir}/preview.html`。从第 1 页开始，点每段提示词右上角的「复制」按钮，把提示词粘贴给 Gemini 或 GPT 让它画图；把生成的 3:4 竖版图片（1080×1440）下载下来，**放进对应笔记的子文件夹**——第 1 篇笔记（post-01）的图放 `{note_dir}/images/post-01/`，第 2 篇放 `{note_dir}/images/post-02/`，以此类推（子文件夹没有就新建，名字和笔记文件名一样）；每个子文件夹里第 1 页命名 `P01.png`、第 2 页命名 `P02.png`……按页码依次命名（序号固定两位数）。全部放好后回复我"图片好了"，我再继续。

等图片就位后再进入图片对抗审查。

两条路径的后续完全一致；绝不在提示词里加二维码/联系方式等违规元素。

**图片对抗审查**：触发 content-reviewer（独立实例）按 `checklist-images.md` 审查——数量/尺寸由脚本核对，错字/风格/人物一致性逐张看图核对。FAIL → 只重出问题页（同一提示词重生成，或请运营重出该页回传），复审通过为止。

### 第 7 步 · 交付提醒（人工发布）

小红书无 API，最后一步始终是人工发布。交付时告诉用户：

- 文件路径与篇数 + `preview.html` 路径 + `images/post-NN/` 各子目录。
- **怎么发**：把「发布文案」整段复制进小红书（含可见标签行）；按 P01→PNN 顺序上传**该篇**子目录 `images/post-NN/` 里的图，首图即封面（别拿错别篇的图）。
- **发布要点**：首图最关键、标签贴够、危机声明保留、不要在任何位置放微信/二维码导流。

## 红线速记（违反任一即不合格）

1. 不新增未验证事实——数字/引语只沿用源 pillar，短文里的每个数字都能在原文找到。**标题/封面里的条数数字必须等于正文实际条数**（说「5 个」就得真有 5 条，不可为凑钩子虚增）。
2. 不用「治疗/诊断/治愈/医院/医生」自我描述；不用极限词；不夸大效果。
3. 任何位置都不放站外导流（微信/二维码/加群/外链）——小红书限流封号红线。
4. 危机声明（全国统一心理援助热线 12356）每篇在位。
