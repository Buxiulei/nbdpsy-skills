# 咨询师推介笔记规格（独立场景，不从 pillar 长文拆分）

> 依据：小红书平台规则、《广告法》§9（极限词）/§17（医疗用语）/§4（虚假宣传）、《交易导流违规管理细则》、全国统一心理援助热线 12356。
> 本场景与「长文拆小红书」共用同一套品牌基底、合规红线与出图/发布链路，仅**取材源不同**：这里取材于官网咨询师公开资料，不拆长文。
> 结构规格、禁用词替换表、全局硬闸 G1–G10 仍以 `references/xiaohongshu-spec.md` 为准，本文只补「咨询师推介」这一场景的差异部分。

一句话定位：把一位咨询师的**后台真实资料**，翻译成一条正在 emo、正在纠结"要不要找咨询师"的真人愿意读下去的图文笔记。卖点是**这个人可信、可托付**，不是"疗效"。

---

## 0. 数据源与字段口径

数据实时从官网免鉴权公开 API 拉取（`scripts/fetch_counselor.py`），**禁止手改**——价格、资质随后台变动，笔记必须与后台一致。

| 字段 | 来源 | 用途 | 口径 / 红线 |
|---|---|---|---|
| `emp_no` | list / 详情 | 咨询师唯一编码，拉详情用 | — |
| `name` / `display_name` | list | 姓名 | 以 `display_name` 为准（`name` 常为空）；`fetch_counselor.py --list` 已做好回退 |
| `title` | list / 详情 | 职称（如「CPS 注册心理师」「北大临床心理硕士」） | 按后台事实原样引用，不升格（别把「助理心理师」写成「资深专家」） |
| `is_accepting` | list / 详情 | 是否接单 | 停单的咨询师**不做推介笔记**（引来了约不上，体验最差）；只给 `is_accepting=true` 的人做 |
| `price_per_session` | list / 详情 | 正式咨询标价 | 展示格式 **「¥N / 次」**（N 直接取整数元，如 `400.0`→「¥400 / 次」） |
| `communication_price` | list / 详情 | 预沟通价 | 展示格式 **「预沟通 ¥N」**；为 `0` 时写 **「预沟通免费」** |
| `specialties` / `profile_sections.specialties` | list / 详情 | 擅长方向 | 只用后台已有条目，**绝不新增擅长领域** |
| `profile_sections` | 详情 | 结构化详细介绍（experience/methods/philosophy/training/testimonials…） | 亮点段的唯一素材库，**引用其原意、不编造** |
| `experience_years` | list / 详情 | 从业年限 | 有则用，无则不提，不估算 |
| `schools` | list | 流派 / 取向 | 有则用（如「精神动力」「CBT」），无则不提 |
| `avatar_url` / `video_intro_url` / `audio_intro_url` | 详情 | 头像 / 音视频介绍 | 仅参考，**不作为发布素材**（真人照片走 §2 授权 + 本地合成） |
| 🔴 `contracted_price` | 详情响应里**曾**出现 | —— | **签约价属用户隐私口径**。`fetch_counselor.py` 已在返回前显式删除该字段；任何情况下**绝不落盘、绝不进笔记、绝不换算展示**。对外价格只有 `price_per_session` 与 `communication_price` 两个。 |

**价格默认不展示（2026-07-24 运营口径定案）**：简历图与介绍文案**默认都不含价格**——大部分推介笔记不露价（先建立信任），文末只留「主页可查预约方式」。**生成前必须问运营一句「这篇要不要展示价格？」**，运营明确说要才加；要展示就只能用上面两个字段的实时值，这是唯一红线。

## 2. 照片三铁律（真人照片入图的强制前提）

推介笔记的 P2 简历卡**可以**放咨询师本人真实照片（比品牌插画更可信）。但真人肖像的合规与失真风险远高于插画，故设三条铁律，任一不满足就**不出图、不发布**：

1. **授权先行**：出图前必须由运营向咨询师本人确认「**已同意将该照片用于小红书宣传**」，并把这句确认回给你。**未拿到确认 = 停等**，不得先斩后奏。授权对象是「这张照片 + 小红书这个用途」，不是泛泛的"同意露脸"。
2. **AI 直出为默认、本人认可为发布闸（2026-07-24 定案）**：默认把授权照片上传图床后作为 `anchor_url` 交给后端 gpt-image-2 **直出带照片的简历卡**（模型会重绘人物——实测还原度很高、版式融合自然，但输出不是原像素）。因此**成图必须经咨询师本人过目认可后才可发布**——运营流程本就让咨询师自审推介内容，把「看成图」并入该环节即可；本人未认可一律不发。咨询师在意像素级真实、或对重绘效果不满意时，改走**备选保真路线**：`scripts/compose_photo.py` 本地几何合成（等比裁剪/圆角/品牌描边，照片不进模型），配 §4 附录的留白底图模板。
3. **照片不入库、用完即弃**：真人照片文件**不提交进 git**（已被 `.gitignore` 之外的产物目录约束，落 `{note_dir}/photo/` 临时用）；交付后提醒运营**自行保管原图**、本地合成产物用完可删。台账里也不留照片。

> 这三条不是流程摆设：一张未经授权或被 AI 改脸的咨询师照片流出，对一个靠信任吃饭的心理工作室是可以上门维权的事故。宁可这篇不配真人照（退回纯插画简历卡），也不碰这三条红线。

## 3. 文案骨架（正文 ~300 字，三段式变体）

沿用 xiaohongshu-spec 的三段式，但主体从「科普干货」换成「**这个人为什么可信**」。**排版同样按 `xiaohongshu-spec.md` §1.6「发布排版规范（手机端实际渲染）」重排**——首行即"选谁"的痛点钩子、每段 2–3 行留空行、强调靠独立成行不靠加粗（小红书正文不渲染 Markdown）、emoji 温和克制、资质分点用 `·`、危机声明单独成段：

| 段 | 占比 | 写什么 | 硬要求 |
|---|---|---|---|
| **① 人群痛点钩子** | 约 20% | 用一个具体场景，让"正在犹豫要不要找咨询师"的人对号入座（如「想找人聊聊，又怕遇到不靠谱的 / 怕被评判 / 不知道选谁」）。 | 具体可感，禁问卷腔；**钩子对准"选谁"的焦虑**，不是某个症状 |
| **② 咨询师亮点** | 约 60% | 从 `profile_sections` 里挑 3–4 个**后台已有**的可信锚点：资质（`title`/证书）、受训背景（`training`）、擅长方向（`specialties`）、咨询风格 / 理念（`philosophy`/`methods`）、从业年限 / 时长（`experience`）。**逐条来自后台资料，绝不编造、不升格。** | 每一句都能在 `--emp` 输出里找到出处；**禁止**「最专业」「最懂你」等极限词；**禁止**「治疗 / 治愈 / 诊断 / 医生」自我描述；「北大硕博」等按后台事实表述、不夸大 |
| **③ 预约轻引导 + 危机声明** | 约 20% | 轻引导：陈述「怎么开始」——可先做预沟通（`communication_price` 口径）、正式咨询价（`price_per_session` 口径，或选择不露价），一句「主页可查 / 站内私信」。危机声明**单独成段**。 | 轻 = 陈述事实给选项，不催不促不承诺；**禁止**站外导流（微信/二维码/加群/外链）；危机声明（12356）单独成段、不与预约 CTA 同句同段（G6）；**G1 最低出口律**：③段至少给一个零门槛动作（如「主页有『第一次咨询会发生什么』长文可先看」「先收藏，想清楚了再约」；预沟通免费时可用免费预沟通充当出口）——预沟通收费的咨询师尤其不能只留付费出口 |

**合规完全沿用 xiaohongshu-spec**：极限词 / 医疗词 / 站外导流 / 疗效承诺全表照旧，发布前用 `scripts/check_compliance.py` 扫一遍每篇 `post-NN.md`。推介场景额外自查两条：① 默认不含价格——若含，须运营明确要求过且数字 = `fetch_counselor.py` 实时值；② 亮点无一句超出后台资料。

## 4. 轮播结构（4–6 页）

文件格式**与长文拆分完全一致**：每页 `### PN · 标题` + `**页面文字**` + 围栏提示词（\`\`\` 块内为可直接喂 Gemini/GPT 的中文提示词），走既有「出图（路线 0 后端一致性出图优先）」链路，页数自检用 `count_xhs.py <post.md> --page-min 4 --page-max 6`（本场景区间 4–6，不带参数默认按长文拆分的 6–9 校验会误判）。品牌基底、文字入图规范、比例参数句式全部沿用 `references/illustration-spec.md`。

- **P1 · 封面**：品牌插画 + 咨询师姓名 / `title` + 一句钩子。**不含真人照片**（封面是品牌门面，用插画）。示例钩子：「想找个人好好聊聊，又怕选错人？」。
- **P2 · 简历信息卡（AI 直出，anchor_url=照片图床直链）**：这是放真人照片的页。**密度硬要求：全卡 8–10 个信息点、2–3 区块满版、图文双通道（每条配一个各不相同的小图标）**——只有姓名+两三条资质的“名片式”简历卡视为不合格，信息全部从 `fetch_counselor.py` 的 profile_sections/字段取材（教育、资质认证、受训经历、执业年限与小时数、擅长方向、咨询风格、适合人群），绝不编造。
  - **提示词骨架**（〇〇替换为该咨询师真实资料；一张装不下就拆 P2 教育资质卡 + P3 方向风格卡两张，都带同一张照片 anchor）：

## 5. 完整流程（每步有验证闸门）

1. **fetch** — `python3 scripts/fetch_counselor.py --list` 选人（或直接 `--emp <emp_no>` 拉详情）。确认 `is_accepting=true`；确认输出里**没有** `contracted_price`。
2. **照片与授权确认（停等闸门）** — 询问运营：这篇要不要放真人照片？要 → 索取照片 + **回传"咨询师本人已同意用于小红书宣传"的确认**；拿不到确认或运营选纯插画 → P2 退回纯插画简历卡（不合成照片）。**未确认不进出图。**
3. **文案** — 按 §3 写 `post-NN.md`（正文 ~300 字 + 可见标签 + `### P1…PN` 轮播 + 页面文字 + 提示词）；亮点逐条对齐 `--emp` 输出。
4. **合规校验** — `python3 scripts/check_compliance.py {note_dir}/post-NN.md` 每篇 `ok=true`；额外核：若含价格须为实时值且运营明确要求过展示（默认不展示）、亮点无一句超出后台资料。
5. **出图** — 插画页按 SKILL「出图」章节常规产出；**简历卡页**：先把授权照片上图床拿直链（`python3 scripts/publish_note.py --upload-images <照片>` 取 urls[0]；7 天过期、目录不可猜，提醒运营照片会短存服务器图床），再 `python3 scripts/gen_images.py --note <post-NN.md> --pages <简历卡页号> --anchor-url <照片直链>` 直出。
6. **本人认可（硬闸）** — 把简历卡成图发给咨询师本人过目，**明确认可后才可进入发布**；不满意可调提示词重出，或改走备选保真路线（compose_photo 本地合成）。
7. **图审** — 逐图核对：图内中文无错字、全套风格一致、简历卡信息点 ≥8 且与后台资料逐条一致、人物与照片观感一致（最终以咨询师本人认可为准）。
8. **发布 / 交付** — 沿用 SKILL 的路线 A（`publish_note.py` 自动发布，**发布前确认账号与篇目**）/ 路线 B（人工发布）；提醒运营真人照片自行保管、不入库。

## 红线速记（推介场景专属，叠加 xiaohongshu-spec §1.5 全局硬闸）
