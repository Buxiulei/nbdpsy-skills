# 咨询师推介笔记规格（独立场景，不从 pillar 长文拆分）

> 依据：小红书平台规则、《广告法》§9（极限词）/§17（医疗用语）/§4（虚假宣传）、《交易导流违规管理细则》、全国统一心理援助热线 12356。
> 本场景与「长文拆小红书」共用同一套品牌基底、合规红线与出图/发布链路，仅**取材源不同**：这里取材于官网咨询师公开资料，不拆长文。
> 结构规格、禁用词替换表、全局硬闸 G1–G10 仍以 `references/xiaohongshu-spec.md` 为准，本文只补「咨询师推介」这一场景的差异部分。

一句话定位：把一位咨询师的**后台真实资料**，翻译成一条正在 emo、正在纠结"要不要找咨询师"的真人愿意读下去的图文笔记。卖点是**这个人可信、可托付**，不是"疗效"。

---

## 0. 数据源与字段口径

数据实时从官网免鉴权公开 API 拉取（`scripts/fetch_counselor.py`），**禁止手改**——价格、资质随后台变动，笔记必须与后台一致。

| 字段 | 来源 | 用途 | 口径 / 红线 |
|---|---|---|---|
| `emp_no` | list / 详情 | 咨询师唯一编码，拉详情用 | — |
| `name` / `display_name` | list | 姓名 | 以 `display_name` 为准（`name` 常为空）；`fetch_counselor.py --list` 已做好回退 |
| `title` | list / 详情 | 职称（如「CPS 注册心理师」「北大临床心理硕士」） | 按后台事实原样引用，不升格（别把「助理心理师」写成「资深专家」） |
| `is_accepting` | list / 详情 | 是否接单 | 停单的咨询师**不做推介笔记**（引来了约不上，体验最差）；只给 `is_accepting=true` 的人做 |
| `price_per_session` | list / 详情 | 正式咨询标价 | 展示格式 **「¥N / 次」**（N 直接取整数元，如 `400.0`→「¥400 / 次」） |
| `communication_price` | list / 详情 | 预沟通价 | 展示格式 **「预沟通 ¥N」**；为 `0` 时写 **「预沟通免费」** |
| `specialties` / `profile_sections.specialties` | list / 详情 | 擅长方向 | 只用后台已有条目，**绝不新增擅长领域** |
| `profile_sections` | 详情 | 结构化详细介绍（experience/methods/philosophy/training/testimonials…） | 亮点段的唯一素材库，**引用其原意、不编造** |
| `experience_years` | list / 详情 | 从业年限 | 有则用，无则不提，不估算 |
| `schools` | list | 流派 / 取向 | 有则用（如「精神动力」「CBT」），无则不提 |
| `avatar_url` / `video_intro_url` / `audio_intro_url` | 详情 | 头像 / 音视频介绍 | 仅参考，**不作为发布素材**（真人照片走 §2 授权 + 本地合成） |
| 🔴 `contracted_price` | 详情响应里**曾**出现 | —— | **签约价属用户隐私口径**。`fetch_counselor.py` 已在返回前显式删除该字段；任何情况下**绝不落盘、绝不进笔记、绝不换算展示**。对外价格只有 `price_per_session` 与 `communication_price` 两个。 |

**价格默认不展示（2026-07-24 运营口径定案）**：简历图与介绍文案**默认都不含价格**——大部分推介笔记不露价（先建立信任），文末只留「主页可查预约方式」。**生成前必须问运营一句「这篇要不要展示价格？」**，运营明确说要才加；要展示就只能用上面两个字段的实时值，这是唯一红线。

## 2. 照片三铁律（真人照片入图的强制前提）

推介笔记的 P2 简历卡**可以**放咨询师本人真实照片（比品牌插画更可信）。但真人肖像的合规与失真风险远高于插画，故设三条铁律，任一不满足就**不出图、不发布**：

1. **授权先行**：出图前必须由运营向咨询师本人确认「**已同意将该照片用于小红书宣传**」，并把这句确认回给你。**未拿到确认 = 停等**，不得先斩后奏。授权对象是「这张照片 + 小红书这个用途」，不是泛泛的"同意露脸"。
2. **AI 直出为默认、本人认可为发布闸（2026-07-24 定案）**：默认把授权照片上传图床后作为 `anchor_url` 交给后端 gpt-image-2 **直出带照片的简历卡**（模型会重绘人物——实测还原度很高、版式融合自然，但输出不是原像素）。因此**成图必须经咨询师本人过目认可后才可发布**——运营流程本就让咨询师自审推介内容，把「看成图」并入该环节即可；本人未认可一律不发。咨询师在意像素级真实、或对重绘效果不满意时，改走**备选保真路线**：`scripts/compose_photo.py` 本地几何合成（等比裁剪/圆角/品牌描边，照片不进模型），配 §4 附录的留白底图模板。
3. **照片不入库、用完即弃**：真人照片文件**不提交进 git**（已被 `.gitignore` 之外的产物目录约束，落 `{note_dir}/photo/` 临时用）；交付后提醒运营**自行保管原图**、本地合成产物用完可删。台账里也不留照片。

> 这三条不是流程摆设：一张未经授权或被 AI 改脸的咨询师照片流出，对一个靠信任吃饭的心理工作室是可以上门维权的事故。宁可这篇不配真人照（退回纯插画简历卡），也不碰这三条红线。

## 3. 文案骨架（正文 400–800 字，「人物直入」三部分）

> **2026-07-24 定案，来自黄安麟篇实战反馈**：推介笔记是**介绍一个人**，不是科普文。此前照搬科普三段式（痛点场景铺垫 →「选咨询师其实是先认识一个人」类导语 → 干货），把介绍一个人写成了栏目文——**整段删掉**。推介骨架改为「**人物直入**」三部分：开头直接把这位咨询师最硬的资质/画像摆出来，主体把擅长议题逐条展开讲透并埋长尾词，结尾说怎么开始。

不再套 xiaohongshu-spec 的科普三段式。**排版仍按 `xiaohongshu-spec.md` §1.6「发布排版规范（手机端实际渲染）」重排**——每段 2–3 行留空行、强调靠独立成行不靠加粗（小红书正文不渲染 Markdown）、emoji 温和克制、资质分点用 `·`。**唯一例外：§1.6「F · 危机声明的排版位置」条对本场景不适用**（本场景不含危机声明，见下）。

| 段 | 占比 | 写什么 | 硬要求 |
|---|---|---|---|
| **① 开头 · 人物直入** | 约 15%（1–2 段） | **姓名 + 最硬的资质 / 画像钩子直接砸出来**（如「黄安麟，北大临床心理硕士，做复杂性创伤（CPTSD）疗愈第 8 年」）。**前 10 字必须立住关键信息**——读者一眼知道"这是谁、硬在哪"。 | **禁止痛点场景铺垫**（「想找人聊聊又怕选错…」这类整段删）；**禁止「选咨询师其实是先认识一个人」类导语**；开头即人，不即"焦虑" |
| **② 主体 · 擅长议题逐条展开 + 流派 + 数据** | 约 70%（核心篇幅） | **(a) 擅长议题逐条展开**：每条 2–4 句——这类议题**是什么人的什么困扰** ＋ **黄安麟/该咨询师会怎么和你工作**（称呼直呼其名，见红线）（取材 `profile_sections` 原意）；**每条至少嵌 1 个长尾关键词**。<br>**(b) 受训流派与工作方式一段**：写出**系统受训的流派名**（CBT / 精神动力 / 整合取向…，取材 `profile_sections` 的 `training`/`methods`）＋ 一句**白话解释它怎么帮到来访者**。<br>**(c) 执业数据**：从业年限 / 累计咨询小时数（`experience_years`/`experience`，有则写、无则不提）。 | 每一句都能在 `--emp` 输出里找到出处；**长尾词只能来自该咨询师后台资料真实覆盖的方向，绝不硬蹭**（黄安麟的原生家庭创伤工作里真的包含 CPTSD，才写 CPTSD）；**禁止**极限词（「最专业」「最懂你」）；**禁止**「治疗/治愈/诊断/医生」自我描述；资质按后台事实表述、不升格 |
| **③ 结尾 · 怎么开始 + 轻互动** | 约 15% | 陈述「怎么开始」——可先做预沟通（`communication_price` 口径）、正式咨询（`price_per_session` 口径，**价格默认不露**，规则见 §0）、一句「主页可查预约方式 / 站内私信」；末尾一句**轻互动**（如「在意的方向评论区告诉我，帮你看看合不合适」）。 | 轻 = 陈述事实给选项，不催不促不承诺；**禁止**站外导流（微信/二维码/加群/外链）；末尾轻互动可给个零门槛选项（「先收藏，想清楚了再约」「主页有『第一次咨询会发生什么』长文可先看」），但**不作 G1 硬闸强求**（G 系适用性见下） |

**长尾关键词埋词表（示例，只在该咨询师后台真实覆盖时用）**：原生家庭创伤 → **CPTSD / 复杂性创伤 / 童年情感忽视**；亲密关系困扰 → **依恋修复 / 追逃循环 / 安全型依恋**；情绪内耗 → **自我关怀 / 反刍思维 / 过度自我批评**；职场压力 → **职业倦怠 / 冒充者综合征**。**议题不能只列干词**（「原生家庭创伤 / 亲密关系困扰 / 情绪内耗」这样一排短语就完事＝不合格）——每类都要展开成"是什么困扰 + 怎么工作"的详细描述，长尾词嵌在描述里。

**展开正例（合格颗粒度示范，2–4 句/条）**：
> 🌱 原生家庭创伤：如果你在一个长期被否定、被忽视的家里长大，成年后那种「说不清哪里坏了、但总觉得自己有问题」的感受，可能与复杂性创伤（CPTSD）有关。黄安麟在这类议题上做了多年工作：不急着翻旧账，而是先陪你把当年的适应方式看清楚，再一点点长出新的应对。

展开到这个颗粒度才算合格：有人群（谁）、有体验描述（对得上号）、嵌了长尾词（CPTSD/复杂性创伤）、有咨询师怎么工作（不是空泛承诺）。

**字数：正文 400–800 字**（比科普笔记的 300 字更长，推介要把人讲透）。自检命令见 §5 第 4 步：`count_xhs.py <post.md> --body-min 400 --body-max 800 --page-min 4 --page-max 6`。

**危机声明：本场景不加**（正文与末页图都不加）。原因：推介是**人物介绍**、不是心理科普内容，在一条"介绍一个人"的笔记里硬塞「本文为心理科普，不构成诊断或治疗…12356」的科普声明，反而显得像在科普栏目里、与"介绍这个人"的调性打架。合规校验因此走 `check_compliance.py <post.md> --no-crisis`（跳过危机声明在位检查，极限词/医疗违禁/站外导流照扫）。

**合规其余照旧沿用 xiaohongshu-spec**：极限词 / 医疗词 / 站外导流 / 疗效承诺全表不放松。推介场景额外自查两条：① 默认不含价格——若含，须运营明确要求过且数字 = `fetch_counselor.py` 实时值；② 亮点与长尾词无一句/一词超出后台资料。

**G 系硬闸适用性（推介场景）**：**禁站外导流（G7）/ 极限词 / 医疗词自我描述 / 不编造（G4 精神：不代读者作结论、只写后台有出处的事实）仍全部适用**；**G1 最低出口律、G6 危机与 CTA 分离对推介场景不适用**——推介不含危机声明（无 G6 可分离的对象），结尾的收藏/评论互动是自然选项而非 G1 强求的"零门槛出口"。若某篇确实要放来访者反馈页，G5 案例三律照旧适用（见 §4）。

**标签（可见 `#标签` 行，3–6 个）**：复用库内原词规则照旧（能对上就用已有标签），**但其中必须含该咨询师核心议题的长尾词**（如 `#CPTSD` `#复杂性创伤` `#依恋修复`）——长尾词是被搜索到的关键，标签里不能只有 `#心理咨询` `#心理科普` 这类泛词。

## 4. 轮播结构（4–6 页）

文件格式**与长文拆分完全一致**：每页 `### PN · 标题` + `**页面文字**` + 围栏提示词（\`\`\` 块内为可直接喂 Gemini/GPT 的中文提示词），走既有「出图（路线 0 后端一致性出图优先）」链路，页数与字数自检用 `count_xhs.py <post.md> --body-min 400 --body-max 800 --page-min 4 --page-max 6`（本场景页数区间 4–6、正文区间 400–800；不带参数默认按长文拆分的 6–9 页 / 210–450 字校验会误判）。品牌基底、文字入图规范、比例参数句式全部沿用 `references/illustration-spec.md`。

- **P1 · 照片封面（AI 直出，与简历卡同一张照片 anchor，2026-07-24 定案）**：封面就放咨询师本人照片——照片人物为主体 + 大字姓名/职称钩子 + 品牌色底。**钩子必须人物直入**（最硬资质/画像浓缩，前 10 字立住信息，如「黄安麟｜北大临床心理硕士」「陪你走出复杂性创伤的第 8 年」），**禁止任何痛点铺垫句**（「想找个人好好聊聊，又怕选错人？」这类=返工——封面是介绍人，不是贩卖焦虑）。
  - 出图：P1 与简历卡页**同批**带同一张照片 `--anchor-url` 直出（人物跨页一致）；封面同样过**咨询师本人过目认可**硬闸；在意像素级真实时封面也可走备选保真路线（compose_photo 照片大图 + 本地文字条）。
  - **封面提示词骨架**：
    ```text

    参考图中这位咨询师的照片为画面主体（保持同一人的样貌、姿态与衣着，占画面约 2/3，暖色柔光、与背景自然融合），背景暖米白 #E8D8C4 与雾霾蓝灰 #A8B5C4 渐层。
    大字标题「〇〇〇」（咨询师姓名，深蓝 #2B3A4A 粗体，画面上部或下部留白处），副题一行「〇〇〇〇〇〇〇〇」（职称/最硬资质浓缩 ≤12 字，同色小一号）。
    图内文字仅此两行，逐字清晰无错字；不出现二维码/微信/电话/站外引流。
    ```
- **P2 · 简历信息卡（AI 直出，anchor_url=照片图床直链）**：这是放真人照片的页。**密度硬要求：全卡 8–10 个信息点、2–3 区块满版、图文双通道（每条配一个各不相同的小图标）**——只有姓名+两三条资质的“名片式”简历卡视为不合格，信息全部从 `fetch_counselor.py` 的 profile_sections/字段取材（教育、资质认证、受训经历、执业年限与小时数、擅长方向、咨询风格、适合人群），绝不编造。**图文分工**：简历卡**图上的议题药丸保持 2–6 字短词**（「原生家庭」「亲密关系」「情绪内耗」——药丸太长会撑破版式、也易出错字），**长尾关键词（CPTSD/复杂性创伤/依恋修复…）由正文文案承载**，不塞进图内药丸。受训流派名（CBT/精神动力/整合取向）可入图（「受训与执业」区块），也在正文写全。
  - **提示词骨架**（〇〇替换为该咨询师真实资料；一张装不下就拆 P2 教育资质卡 + P3 方向风格卡两张，都带同一张照片 anchor）：

## 5. 完整流程（每步有验证闸门）

1. **fetch** — `python3 scripts/fetch_counselor.py --list` 选人（或直接 `--emp <emp_no>` 拉详情）。确认 `is_accepting=true`；确认输出里**没有** `contracted_price`。
2. **照片与授权确认（停等闸门）** — 询问运营：这篇要不要放真人照片？要 → 索取照片 + **回传"咨询师本人已同意用于小红书宣传"的确认**；拿不到确认或运营选纯插画 → P2 退回纯插画简历卡（不合成照片）。**未确认不进出图。**
3. **文案** — 按 §3「人物直入」三部分写 `post-NN.md`（正文 400–800 字 + 可见标签含长尾词 + `### P1…PN` 轮播 + 页面文字 + 提示词）；擅长议题逐条展开并嵌长尾词、受训流派写明、亮点逐条对齐 `--emp` 输出。
4. **合规校验 + 字数页数自检** — `python3 scripts/check_compliance.py {note_dir}/post-NN.md --no-crisis` 每篇 `ok=true`（`--no-crisis` 跳过危机声明在位检查——本场景不含 12356 声明；极限词/医疗违禁/站外导流照扫）；`python3 scripts/count_xhs.py {note_dir}/post-NN.md --body-min 400 --body-max 800 --page-min 4 --page-max 6` 每篇 `ok=true`。额外核：若含价格须为实时值且运营明确要求过展示（默认不展示）、亮点与长尾词无一句/一词超出后台资料。
5. **出图** — 插画页按 SKILL「出图」章节常规产出；**简历卡页**：先把授权照片上图床拿直链（`python3 scripts/publish_note.py --upload-images <照片>` 取 urls[0]；7 天过期、目录不可猜，提醒运营照片会短存服务器图床），再 `python3 scripts/gen_images.py --note <post-NN.md> --pages <简历卡页号> --anchor-url <照片直链>` 直出。
6. **本人认可（硬闸）** — 把简历卡成图发给咨询师本人过目，**明确认可后才可进入发布**；不满意可调提示词重出，或改走备选保真路线（compose_photo 本地合成）。**实操通道**：简历卡成图用 `python3 scripts/publish_note.py --upload-images <P2成图.png>` 上图床，把返回的 url 发运营转给咨询师本人过目——链接点开即看、无需来回传文件（7 天过期，过期重传）。⚠️ **务必区分两种链接**：第 5 步为出图上传的**授权照片原图**图床链接**只作 anchor_url 喂后端生图用，绝不当查看链接转发扩散**（那是咨询师真人肖像原图，隐私红线）；转给本人过目的只能是**已重绘/合成的成图**链接。
7. **图审** — 逐图核对：图内中文无错字、全套风格一致、简历卡信息点 ≥8 且与后台资料逐条一致、人物与照片观感一致（最终以咨询师本人认可为准）。
8. **发布 / 交付** — 沿用 SKILL 的路线 A（`publish_note.py` 自动发布，**发布前确认账号与篇目**）/ 路线 B（人工发布）；提醒运营真人照片自行保管、不入库。

## 红线速记（推介场景专属，叠加 xiaohongshu-spec §1.5 全局硬闸）

0. **称呼直呼其名（2026-07-24 定案）**：正文、图内文字、标题提及咨询师一律直呼其名（「黄安麟」）或用「她/他/TA」，**绝不加「老师」「X 老师」「咨询师老师」等称谓**（用户对话里说「XX 老师」不影响，产出内容里不出现）。
1. `contracted_price` 永不出现在任何产物里；对外价格只有 `price_per_session`（¥N / 次）与 `communication_price`（预沟通 ¥N / 免费）两个，取实时值不手改；**价格默认不上笔记不上图，生成前问运营、明确要求才展示**。
2. 亮点每一句都能在 `fetch_counselor.py --emp` 输出里找到出处；资质按后台事实表述，不升格、不编造擅长领域。
3. 真人照片：**授权先行 → AI 直出成图须咨询师本人认可后才发（备选：本地合成保真）→ 不入 git**，任一环节过不去即退回纯插画。
4. 只给 `is_accepting=true` 的咨询师做推介；不用医疗词自我描述、不用极限词、不承诺疗效、不做站外导流。**本场景不加危机声明**（推介是人物介绍非科普内容，正文与末页图都不放 12356；合规校验走 `check_compliance.py --no-crisis`）。
5. **议题必展开 + 埋长尾词 + 写流派 + 字数够长**：擅长议题逐条展开成"是什么困扰 + 怎么工作"、每类嵌 ≥1 个真实覆盖的长尾词（原生家庭创伤→CPTSD/复杂性创伤…）、受训流派（CBT/精神动力/整合取向）写出来、正文 400–800 字；标签 3–6 个里必含核心议题长尾词。长尾词绝不硬蹭后台没有的方向。
