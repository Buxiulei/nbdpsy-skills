# 小红书图文笔记创作（从 pillar 长文拆分）

承接 `nbdpsy-seo-artical-creator` 产出的心理科普 pillar 长文，把一篇 4000+ 字的长文拆成 **5–8 条可直接发小红书的图文笔记**。每条笔记 = 一段 **~300 字正文** + 一套 **6–9 页"PPT 式"轮播插图**（每页给出"页面文字"+"中文绘图提示词"，提示词把要显示在图上的中文直接写进去，复制即可喂给 Gemini/GPT 出图）。

**项目背景一句话**：NBDpsy 是咨询师全员北大硕博的纯线上华人心理咨询工作室。官网 pillar 长文负责被搜索引擎/AI 引擎收录；小红书负责触达正在 emo、正在内耗、正在搜"怎么办"的真人。长文重信源与循证，小红书重共情与钩子——本 skill 的使命是把前者的可信内容，翻译成后者的语言，**绝不在翻译中编造新事实**。

## 三条贯穿全程的铁律（理解动机，别只背规则）

1. **不新增未验证事实**。统计数字、专家引语、出处只能沿用 pillar 原文里已核实过的内容。短文里出现的每个数字，都要能在源长文里找到。小红书没有参考文献区，但"编造一个数据被扒出来"对一个心理工作室是致命的——所以宁可少写一个数字，不可造一个。
2. **合规红线高于传播效果**。心理/健康类是小红书重点审核领域：极限词（最/第一/根治/100%/彻底）违广告法、医疗词（治疗/治愈/诊断）非医疗机构禁用、站外导流（微信/二维码/加群）直接限流封号。详见 `references/xiaohongshu-spec.md`，发布前用脚本扫一遍。
3. **发布走双轨：自动优先、人工兜底**。配图按「出图（后端一致性出图优先，宿主自适应兜底）」章节产出（凭据在位时首选后端 gpt-image 锚点法自动出图，缺凭据才由宿主自动生成或人工把每页提示词喂给 Gemini/GPT）。发布环节：若本机配了 `NBDPSY_XHS_API_KEY`（管理员发的运营接入包带此凭据），用 `scripts/publish_note.py` 经 nbdpsy-api（自建 REST 服务）自动发布——**发布前必须经运营确认账号与篇目**；没配凭据则回到人工发布，交付时把"怎么发"讲清楚。

多步任务先维护一份任务清单，每步勾掉。

> **路径约定**：以下 `{SKILL_DIR}` 指本 SKILL.md 所在目录；`{workspace}` 指内容工作区（实际路径用 `python3 {SKILL_DIR}/scripts/nbdpsy_common.py workspace` 查询）；`{note_dir}` 指一次拆分的产物目录 `{workspace}/xiaohongshu/{slug}/`。

## 完整流程（0–7 步，每步都有验证闸门）

```
0. 定位源长文               → 验证：拿到源长文的本地 md 路径（fetch_post.py 输出的 path）
1. 规划拆分                 → 验证：5–8 篇（用户指定篇数时以用户为准），每篇一个独立钩子、彼此不重叠
2. 写正文（每篇 ~300 字）    → 验证：count_xhs.py 字数达标 + 钩子/分点/CTA 齐 + 不新增未验证事实
3. 设计配图轮播（6–9 页）    → 验证：每页复用固定品牌基底 + 图中文字短 + 首页是钩子封面 + 视频参考图提示词页对页齐
4. 合规校验                 → 验证：check_compliance.py 逐篇全绿 + 危机声明每篇在位 + 语义自检过
5. 落盘 + 预览页 + 笔记审查  → 验证：一篇一文件写入 {note_dir}，含 00-overview 与 {note_dir名}-preview.html；笔记对抗审查 PASS
6. 出图（宿主自适应）+ 图审  → 验证：{note_dir}/images/post-NN/P01…PNN.png 齐全；图片对抗审查 PASS
7. 发布（自动可选）或交付     → 验证：自动=publish_note.py 轮询到 published 拿到 note_url；人工=给出文件路径 + 发布要点（首图最重要/3:4/标签/危机声明保留）
```

### 第 0 步 · 定位源长文

按输入类型三选一：

- **用户给了 slug** → `python3 {SKILL_DIR}/scripts/fetch_post.py --slug <slug>`：自动从官网公开 API 拉取全文，落到 `{workspace}/drafts/{slug}.md`（stdout 输出 JSON，`path` 字段即文件路径；本地已有同名文件直接复用缓存、不打网络。要落到别处加 `--out <目录>`）。
- **用户没给** → `python3 {SKILL_DIR}/scripts/fetch_post.py --list 10` 列最新文章（JSON 数组：slug/标题/发布时间）让用户选；`--list` 不带数字默认 10。全自动流水线中直接用上一级刚发布的 slug，无需询问。
- **用户直接给了本地文件路径** → 直接用。

拿到文件后先把全文读完——尤其是 TL;DR、各 H2、对比表、统计块、FAQ，这些是拆分的原料。

### 第 1 步 · 规划拆分 —— **按「痛点场景」分箱，不是按知识点切段**

> ⛔ **本步是整条产线的生死线**（详见 `references/xiaohongshu-spec.md` §1.5，动手前必读那一节）。
>
> **停留在现象层的笔记，读者反应是"学到了"（收藏→划走→忘记）；深挖到场景层的笔记，读者反应是"这说的就是我"（对号入座→情绪被击中→才可能付费）。付费不是被科普说服的，是被场景击中的。**

**长文是知识组织（按主题/机制分 H2），笔记必须是场景组织。**所以拆分不是切段落，是**换一套坐标系重新分箱**：

1. 先列出长文里的知识点/机制/方法（原料）。
2. 再问：**「这个知识点最能救的是哪个具体场景里的哪个人？」**
3. **一篇笔记 = 一个具体场景 ＋ 长文里能解答这个场景的那部分知识**（同一知识点可服务多个场景，不必和 H2 一一对应）。
4. 5–8 篇 = 5–8 个**互不重叠的具体场景**，不是 5–8 个知识角度。

**但"被击中"只是必要条件**：一个已经痛到极点、却被"我不好意思 / 我试过没用 / 我不知道找谁 / 我一直没约"钉住的读者，你再给他一篇更痛的场景文，他只会**更痛地划走**。所以每篇还要回答第二个问题——**它把读者往前推了哪一格？** 那一格就是它的**深挖路径（主路径，唯一）**。

**深挖路径清单（12 条 · 详情与护栏见 spec §1.5，动手前必读）**：

| 层 | 路径 | 公式（一行） |
|---|---|---|
| 认知 | **P1 痛感认领** ≤1 | 白描处境 ＋ "你不用解释" ＋ 人群/慢性度锚点 ＋ 一个今天能做的小动作 |
| 认知 | **P2 处境命名** ★主力 | 具体处境（谁+时刻+那句话/那个身体反应）＋ **计数器**（第几次了）＋ 给它一个**议题名**（不是诊断名） |
| 认知 | **P3 归因矫正** ≤1 | 你以为是 X ＋ 其实更可能是 Y（**机制，不是病名**）＋ 可自验线索 ＋ 下一步 |
| 阻力 | **P4 代价显影** ⛔ | **生产配额 = 0**。落进 P4 = 打回（改写成 P2 慢性版）。分类保留只为当废稿检测器 |
| 阻力 | **P5 去羞耻化** | 把"性格缺陷"重述为"当年的适应"（**条件式**）＋ 常见性（不编数字）＋ 一句许可 |
| 阻力 | **P6 失效归因** ≤1 | 你试过的 X ＋ 它为什么失效（机制，不贬低）＋ 什么条件下才有效 ＋ 你造不出的那部分 |
| 阻力 | **P7 价值论证** ≤1 | 免费替代品（AI/闺蜜/书）真实有效的部分 ＋ 它结构上做不到的那件事 ＋ **诚实分界：什么时候它真的够了** |
| 转化 | **P8 求助路径显影** | 怎么挑咨询师 / 自助 vs 专业的分界 / 换人不是失败（**⛔ 疗法对比与服药并行 = F1，公域冻结**） |
| 转化 | **P9 首访祛魅** ★打头阵 | 三子形态**选一个做落点**：a 过程时间线 / b **保密与记录** / c 表达门槛（"你不需要说得很惨"） |
| 转化 | **P10 可得性破解** ≤1 | 结构约束（时差/母语/本地供给）＋ 怎么绕过 ＋ 线上做不到什么（**主战场在搜索场，小红书只试水**） |
| 转化 | **P11 临门推动** ≤1 | 把"去咨询"切成一个 5 分钟的动作 ＋ 一个时间锚（**禁限时/名额，那是促销**） |
| 关系人轴 | **PX 带他进门** ≤1 | 读者 ≠ 受苦者时走这条：落点**必须是读者自己能做的动作**，绝不能是"如何让对方改变"（他者解码 = 毙） |

**选型一句话**：**看落点——"读完这篇，作者希望读者完成的那一个内在动作是什么？"（看最后三行）** 数出唯一的那一个，就是主路径；数不出唯一一个 = 选题没想清楚，返工。**不做序号遍历**（读者会跳闸），层内并列时取**客户质量更高、合规风险更低**的那条。

**配比一句话**：7 篇 = **固定 5 篇（P2×2 · P5 · P9 · P8）+ 弹性 2 篇（P1/P3/P6/P7/P10/P11/PX 中选）**，**P4 = 0、F1 不进公域**；硬约束：**转化层 ≥1 且必含 P9**、任一路径 ≤2（**P2 可到 3**）、**P3/P4 不得作次要路径**、护栏与配额按"篇内出现的任一路径元素"计（不按主路径计）。

**篇数以用户为准（1–2 篇小批量模式）**：用户明确指定篇数（「只拆 1 篇」「出 2 条」）时按用户说的来——不凑 5–8、不执行上面的固定配比。小批量选型优先级：**单篇选 P2 处境命名**（主力路径，最容易被击中；用户点名了具体场景/角度则以其为准），**两篇选 P2 + P9 首访祛魅**（一篇击中场景＋一篇降求助门槛，认知层+转化层的最小闭环）。**其余护栏一律不降**：五问、慢性度计数器、G1–G10、危机声明、合规扫描照常，P4/F1 依旧零产出。`00-overview.md` 照写（条目少而已），末尾注明**本次未覆盖的原文核心点**，方便下次从同一篇长文续拆时不撞车。

**场景深挖五问**（每篇动笔前逐题回答，答不上来 = 还没挖够，禁止开写）：
① **谁**（具体到能在脑子里看见这个人：产后 6 个月、婆婆天天在家、丈夫说你想太多的新手妈妈）→ ② **什么时刻**（凌晨三点 / 周日下午四点 / 家长会散场那一刻）→ ③ **正在发生什么冲突或代价**（睡不着 / 不敢开口 / 关系在冷掉）→ ④ **这一幕反复到第几次了**（慢性度计数器："第十七个周日"；**单点急性 → 钩来的是买止痛就走的脱落客，加计数器改写或换题**）→ ⑤ **她自己为什么解决不了**（试过什么、**机制上**为什么无效、哪一部分她一个人造不出来）——⑤ 同时必须能回答反面：**什么情况下她自助其实就够了**（诚实分界）。
> ⚠️ 旧第 ④ 问「不管它会怎样」已删除：它直接生产 P4 代价显影（本版禁产），且**恐惧不是求助的独立预测因子**——决定客户质量的是**慢性度**，不是痛感强度。

把拆分大纲写进 `00-overview.md`，每篇必须写全：序号 / **主路径（唯一）+ 次要路径（≤1，不得填 P3/P4）** / **受众（当事人 or 身边人）** / **慢性度** / **场景一句话**（含处境+冲突）/ **五问答案** / 钩子标题 / 对应原文哪些 H2 / 核心 takeaway / 拟用统计或引语。

⚠️ **合规护栏（YMYL 红线，十条全局硬闸见 spec §1.5「G1–G10」）**：深挖场景 ≠ 制造恐慌。**G1 最低出口律**（每篇末必须有一个今天能做、门槛低于付费的小动作）、**G2 躯体分流律**（写到胃疼/失眠/心悸必须同段落说"先排除器质性原因"）、**G3 不给第三人贴标签**（禁 NPD/PUA/回避型式给读者的父母伴侣孩子下判断）、**G4 不代读者作结论**（成因一律条件式）、**G5 案例三律**（书面同意或复合案例并明示 + 禁可反向识别组合）、**G6 危机与 CTA 分离**、**G10 不卖依赖**（收尾指向"有一天你不再需要它"）。风险提示必须在源长文有出处；高危议题（自伤/自杀/重度精神障碍）只做识别与转介（12356），**绝不暗示我们能处理危机**。判据：读者看完应感到「被理解了，且知道下一步能做什么」，不是「被吓到了，必须马上花钱」。

- [ ] **一票否决**：每个选题换成别的行业/人群还说得通吗？（"孩子内向不爱说话"教育机构也能写 = 还在现象层 → 返工）
- [ ] **主路径唯一且已标注**：每篇 frontmatter 与 `00-overview.md` 都有 `scene_path`（唯一一条）；写不出唯一落点 = 返工。次要路径 ≤1 且**不是 P3/P4**
- [ ] **配比合理**（常规 5–8 篇；用户指定 1–2 篇时按上面「小批量模式」选型，不按本条配比）：转化层（P8/P9/P10/P11）≥1 篇**且含 P9**；任一路径 ≤2（P2 可到 3）；**P4 = 0 篇、F1 零出现**；落点互不撞车

- [ ] 5–8 篇并排读起来像**5–8 个不同的人在不同处境里求救**，而不是像一份目录（第一章定义、第二章区别、第三章方法 → 返工）

### 第 2 步 · 写发布文案（正文 ~300 字 + 可见标签）

每篇 post 的第一个内容块就是**复制即发的"发布文案"单元**——这是用户真正粘进小红书输入框的东西。块标题用 `## 发布文案（复制这一段直接发布）`，里面依次是：① 正文主体 ② 危机声明一行 ③ **可见的 `#标签` 行**。

> **标签必须可见**：标签除了写进 frontmatter 的 `hashtags`（供机器读取），**必须同时作为可见文本 `#标签1 #标签2 …` 附在文案末尾**。frontmatter 在网页/编辑器渲染时不显示，只放 frontmatter 等于「看不到标签、也没法复制」。小红书惯例标签贴在正文末尾。

打开 `references/xiaohongshu-spec.md` 对照写。

**正文必须是三段式骨架（强制，不是建议）**：

> **① 痛点场景开头（15–20%）→ ② 科普干货主体（65–75%）→ ③ 结尾轻引导（1–2 句）**
>
> **为什么**：纯科普易涨粉、难转化（读者收藏完就走）；硬广易转化、难曝光（限流+投流拒审+读者反感）。三段式是唯一的解——它对齐小红书真实决策链路（痛点搜索 → 案例对比 → 私信咨询 → 社群成交），也符合投流过审规律（**全文绝大部分是科普干货、只有结尾一句轻引导的笔记最容易过审**；通篇引导或开头即广告的，限流与拒审风险最高）。

- **① 痛点场景开头**：**直接把第 1 步挖出的那个场景写出来**（五问的 ①②③ ——谁、什么时刻、正在发生什么），让读者对号入座（"凌晨三点又醒了，脑子里循环白天那句话"）。首句即 3 秒钩子。禁用"你是否常常感到焦虑？"这种问卷腔。**开头写不具体 = 第 1 步的场景没挖够，回去重挖，别在这里硬凑。**
- **② 科普干货主体**：笔记的价值来源——讲机制、给判据、给可执行方法，分点写。**这一段完全不谈自家服务**；干货要能独立成立（读者不点主页也有收获），数字/结论可溯源到源长文。
- **③ 结尾轻引导**：先给**一个今天能做、门槛低于付费的小动作**（G1 最低出口律，全路径适用——只共情不给出口 = 高唤起零效能，读者会关掉手机而不是求助），再 CTA（提问引评论 /「收藏不迷路」/「评论区抱抱」），再**轻**提一句"还有专业支持这条路"。**轻 = 陈述事实、给出选项，不催不促不承诺**。安全句式与反面例子见 spec §1「轻引导写法」；**结尾的最后三行 = 这篇的落点，必须能读出唯一一条主路径**（选错落点 = 主路径标注作废，返工）。品牌锚句每篇只出现一次，自然带过。
- ⛔ **结尾禁止硬广特征**：限时/优惠/立减/名额/仅剩/立即/私信我领——这些词会导致投流拒审 + 自然流限流（脚本会扫，见第 4 步）。更禁止站外导流（微信/二维码/外链，直接封号）。

其它骨架记忆点：

- **口吻**：第一人称共情 + 干货清单 + 自救指南，口语化，像朋友说话。**共情先于科普**（这是心理内容区别于普通种草文的关键）。每段 2–4 行、配 1 个克制的 emoji（治愈系不要满屏）。
- **标题 ≤20 字，必须承载场景而不是主题**：标题是场景的浓缩，不是知识点的命名。
  - ❌「CPTSD 和 PTSD 的区别」（知识点命名，读者不觉得关自己事）
  - ✅「一句'方案再改改'，我在工位上僵住了」（场景，读者秒对号）
  - 四类钩子（数字体/痛点体/悬念体/身份标签）任选，但**钩子里要有那个场景的影子**。
- **标签 5–10 个**：关联（精准品类如 #心理科普 #CPTSD）+ 热度（蹭当下）+ 长尾（细分/身份/地域，如 #留学生心理 #情绪内耗）。同时写进 frontmatter `hashtags` 与文案末尾可见标签行。

**字数默认 300 汉字**（用户指定值，只数正文主体、不含标签）。三段式在 300 字下的典型配比：开头 50–60 字 / 主体 200–220 字 / 结尾 30–40 字。提醒：小红书爆款正文常在 500–800 字、信息更厚更易被收藏；如想提收藏率可把默认调到 500–600，但默认遵守 300（注意：count_xhs.py 的判定区间固定按 300 基准 210–450，调高目标字数时以人工核对为准）。

**验证**：`python3 {SKILL_DIR}/scripts/count_xhs.py <post.md>` 输出 JSON——`body_chars` 只数「发布文案」区块的汉字（剔除标签行），区间 210–450；`pages` 同时核轮播页数 6–9；以 `ok` 为 true 为过。此外：**三段式在位（痛点场景开头 / 科普主体不谈服务 / 结尾轻引导 1–2 句且无硬广词）**；发布文案块含可见标签行；第 1 步规划的钩子有落地；正文里的数字都能在源长文找到出处。

### 第 3 步 · 设计配图轮播（每篇 6–9 页）

每条笔记配一套"像 PPT 一样"的轮播图。**核心规则见 `references/illustration-spec.md`**，记忆点：

- **每页 = 一张"知识卡"**：① 页面文字 ② 中文绘图提示词。**封面/末页精简**（大标题 + 一句）；**内容页要高密度——但密度 = 图文合起来表达的信息量，不是字数**：把每条信息画成一个能看懂的**具体场景**、抽象点落成具体画面（"有周期"→情绪曲线、"至少两周"→日历、"成组出现"→并列几个症状小人），一页 6–10 个信息点满版铺开。
- **内容页 = 图文双通道（对齐参考图，非纯文字列表）**：**硬要求只有一条——每条/每格/每节点都配一个各不相同、把该点画成具体场景的手绘小图或简笔小人**（图在叙事不在装饰、文字在说明，两者合起来才是密度），配合满版、分 2–3 区块、一页 6–10 信息点。版式按内容选（**17 种，一套 6–9 页混用 3–5 种、别整本都对比表**）——分区图标清单/对比表/流程步骤/象限/隐喻/数据占比/拆解结构/因果链/时间轴周期/自查清单/恶性循环圈/情景对话/身体地图/误区纠错/换个想法(认知重构)/程度光谱/概念定义卡；选型第一问"这页信息之间是什么关系"（映射表见 illustration-spec 第 3 节）；**套用时按金标准范例的详细度展开**——每种版式都有一条实测级完整范例在 `references/layout-gallery.md`，先抄对应范例的结构与颗粒度再替换主题内容（`〇〇` 占位换成具体内容 + 每个元素写成具体画面，别把骨架直接丢给模型）。⚠️ 隐喻图/拆解图别只放"一个大隐喻+几个抽象标签"（低密度陷阱，把每部分落成具体场景或补足信息点）。纸质暖框、粗黑标题、赭红高亮等都是**可选**设计项（标题不必粗黑，默认沿用品牌柔和标题即可）。判据与模板见 `references/illustration-spec.md` 第 1、3 节。
- **图中文字直接写进提示词**：你用的 Gemini/GPT 能渲染中文，所以把这页要显示的中文用引号写进提示词，并指定位置与层级。每个文字块（标题/分区标题/图标标注/说明）各自要短、独立成行——渲染更准、也更好扫；**信息图标注点位多、错字风险更高**，点位式短标注 ≤6 字、关键术语放大，出图后逐字核对。
- **首页 = 钩子封面**（最重要，决定点击率，不堆要点留悬念，保留柔和治愈单主体）；中间页 = 高密度信息海报（一页讲透一个点）；末页 = 行动建议 + 品牌一句话 + 危机声明。
- **全套复用固定品牌调色板与人物卡**：封面/末页/隐喻页用柔和治愈单主体基底；内容页用知识海报基底；两类页共用同一调色板（主色 #A8B5C4 / 暖米 #E8D8C4 / 鼠尾草绿 #C9D6CE；可选强调赭红 #A34B3A + 标题深色 #2B3A4A）与固定人物卡，保证 6–9 张风格一致、整个号调性统一。
- **每篇笔记末尾追加 `## 视频参考图提示词` 节**（页对页的"去文字版"提示词）：每页一条 = 该页绘图提示词**剔除所有「图内文字」指令**，保留品牌基底与人物卡，末尾追加「画面中无任何文字」。用途：nbdpsy-text-to-video 图生模式拿这套提示词出"无文字同款图"当每镜首帧——视频会把旁白烧录成字幕，图内若再有文字会和字幕打架。本 skill 只产这节提示词，不在本级出这套图（由 nbdpsy-text-to-video 产线按需生成）。**节内每页用加粗标记 `**P1**`…`**PN**` + 围栏提示词，绝不用 `### PN` 标题**——`### PN` 是配图轮播的页数计数契约，重复使用会让 count_xhs.py 把页数翻倍误判。完整规格与模板见 `references/illustration-spec.md` 第 6 节。

**验证**：每页提示词都含品牌基底；图中文字够短；首页是钩子封面而非内页；页数落在 6–9；视频参考图提示词与轮播页一一对应且不含任何图内文字指令。

### 第 7 步 · 发布（自动可选）或交付（人工兜底）

先探测本机有没有小红书发布凭据：`python3 {SKILL_DIR}/scripts/nbdpsy_common.py secret ensure NBDPSY_XHS_API_KEY`
（无输出 = 已配置；输出键名 = 缺。**绝不用 `secret get` 探测**——它会把密钥值打进 stdout/对话转录）。

**路线 A · 自动发布（有凭据，经 nbdpsy-api）**：

1. **发布前必须经运营确认**（发布是对外动作，不可静默执行）：列出可用账号
   `python3 {SKILL_DIR}/scripts/publish_note.py --list-accounts`，问清 **发到哪个账号、发哪几篇、
   立即发还是定时**（定时时间要带时区偏移，如 `2026-07-14T09:00:00+08:00`）。
2. 逐篇发布（脚本自动读「发布文案」块 + `images/post-NN/` 配图，**发布是异步**，默认轮询到终态）：

   ```bash
   python3 {SKILL_DIR}/scripts/publish_note.py --note {note_dir}/post-01.md --account <账号名或id>
   ```

   stdout JSON：`outcome=published` 时带 `note_url`（回给运营）；`failed` 带 `error`；
   `outcome=unknown` 表示**任务已入队但状态没确认**（网络抖动等）——**绝不重发**，按 hint 用
   `--job <id>` 复查到终态为止；轮询超时仍在跑同理。多篇逐条串行发，别并发轰同一账号。
3. **失败排障**：
   - 报 `Host not allowed` / proxy blocked / 恒超时 → Claude 沙盒拦网：先跑
     `python3 {SKILL_DIR}/scripts/nbdpsy_common.py sandbox allow` 写入放行名单，提醒运营**重启
     Claude Code** 后重试；单次命令也可用 Bash 工具参数 `dangerouslyDisableSandbox` 兜底。
   - `401` → apikey 失效，找管理员轮换重发接入包；`403` → 该账号未授权给这位运营。
   - `error` 提示 cookie 失效/登录 → 按「小红书账号接入与管理」一节走：`--check-cookie` 验活、
     `--extension-info` + 无痕窗扫码 + `--wait-login` 重登（五态中 `error` 是基础设施失败
     ≠ cookie 失效，别急着让人重登）。
4. 发布成功后把每篇的 `note_url` 汇总回报运营。
5. **发错了 / 要改时间**：`--list-jobs` 查到 pending 任务后，用 `--reschedule <id> --schedule <新时间|now>`
   改定时（`now`=转立即发）或 `--cancel <id>` 撤稿——**仅 pending 任务有效**，已在发/已终态改不动。

**路线 B · 人工发布（无凭据，或运营点名要手发）**，交付时告诉用户：

- 文件路径与篇数 + 预览页（`{note_dir目录名}-preview.html`）路径 + `images/post-NN/` 各子目录。
- **怎么发**：把「发布文案」整段复制进小红书（含可见标签行）；按 P01→PNN 顺序上传**该篇**子目录 `images/post-NN/` 里的图，首图即封面（别拿错别篇的图）。
- **发布要点**：首图最关键、标签贴够、危机声明保留、不要在任何位置放微信/二维码导流。

## 小红书账号接入与管理（发布前的自检与运营指导）

发布走的是 nbdpsy-api，账号登录态靠 **chrome 插件**维护（登录没有 API）。用户问「怎么接入 /
怎么登录新账号 / 账号失效了 / 我有哪些号」或路线 A 发布报账号类错误时，按本节处理。

### 接入自检（用户说「帮我做接入自检 / 我配好了吗」时直接跑这个）

**一键自检**（连通性 + 身份 + 被授权账号 + 就绪，可反复跑）：

```bash
python3 {SKILL_DIR}/scripts/publish_note.py --self-check
```

读输出 JSON：`ok=true, ready=true` → 可以发布；`error` 提 `Host not allowed`/超时 → 沙盒拦网
（跑 `nbdpsy_common.py sandbox allow` 后重启 Claude）；`401` → apikey 失效找管理员重发；
`need_relogin` 非空 → 那些号要重新扫码（见下）。凭据本身是否导入用
`nbdpsy_common.py doctor` 看 `xhs_ready`（**别用 `secret get` 探测，会回显密钥**）。

**插件判据**（本机测不到浏览器装没装插件，用状态倒推）：`--self-check` 返回账号数为 0、或
全部 `cookie_status=unknown` 且从没登录过 → 大概率插件还没装/没用过，走下面「装插件」；
有号但 `invalid`/`captcha` → 插件在但登录态过期，走「重新扫码」。

### 装插件（细致指导，逐步带运营做）

跑 `python3 {SKILL_DIR}/scripts/publish_note.py --extension-info`，返回
`download_url`（免鉴权直接下）、`version`、`install_steps`（官方步骤数组）、`server_time`。
把步骤**翻译成人话逐条发给运营**（默认口径，以 install_steps 为准）：

1. 打开 `download_url` 下载插件包 zip，解压到一个固定文件夹（别放回收站/临时目录）。
2. Chrome 地址栏输入 `chrome://extensions` → 右上角打开「开发者模式」→「加载已解压的扩展程序」
   → 选刚才解压的文件夹。
3. 点开插件弹窗，填**两样**（缺 serverUrl 插件连不上）：
   - **serverUrl** = `https://mcp.nbdpsy.com`（本服务地址，即凭据 NBDPSY_XHS_API_BASE）
   - **apikey** = 接入包里那把（本机凭据库键名 NBDPSY_XHS_API_KEY；提醒运营别外传）
4. 进插件详情页勾选「**在无痕模式下启用**」（下一步登录用无痕窗，不勾开不了）。
5. 装好后按下面「登录新账号」扫码。

### 登录新账号 / 重新扫码（登录态过期）

1. **先**跑 `--extension-info` 记下 `server_time`（必须在扫码**之前**取，它是轮询起点）。
2. 让运营：开一个**无痕窗口**（插件账号页上有入口）打开小红书 → 手机小红书 App 扫码登录。
   登录成功后插件会自动把 cookie 推回服务端、账号自动入库。
3. 你轮询确认：`python3 {SKILL_DIR}/scripts/publish_note.py --wait-login --since <server_time>`
   （重登旧号加 `--account-id <id>`）。`done=true` 即完成；新号记得让管理员授权给这位运营
   （首次导入者自动有权）。
4. 验活兜底：`python3 {SKILL_DIR}/scripts/publish_note.py --check-cookie <账号名或id>`——
   五态里 `valid` 才算好；`invalid` 重新扫码；`captcha` 要人工过验证码；
   **`error` 是服务端基础设施失败 ≠ cookie 失效，别让运营白扫一遍，稍后重测**。

### 用某个账号打开小红书（人工运营场景）

插件的账号卡片上有「打开小红书」：它会用服务端存的该号 cookie 开一个已登录的无痕窗口
（评论互动、手动检查笔记都走这里），卡片上也有「检测」按钮 = 手动触发 cookie 验活。
告诉运营用插件操作即可，这一步没有命令行。

## 红线速记（违反任一即不合格）

1. 不新增未验证事实——数字/引语只沿用源 pillar，短文里的每个数字都能在原文找到。**标题/封面里的条数数字必须等于正文实际条数**（说「5 个」就得真有 5 条，不可为凑钩子虚增）。
2. 不用「治疗/诊断/治愈/医院/医生」自我描述；不用极限词；不夸大效果。
3. 任何位置都不放站外导流（微信/二维码/加群/外链）——小红书限流封号红线。
4. 危机声明（全国统一心理援助热线 12356）每篇在位，且**单独成段、不与商业 CTA 同句同段**（G6）。
5. 全套轮播图复用同一品牌基底，风格一致；图中文字短、不编造专业名词拼写。
6. **主路径唯一且已标注**（`scene_path`）；**P4 代价显影零产出**、**F1 疗法对比/服药并行零出现**；**不给读者的父母/伴侣/孩子贴标签**（NPD/PUA/回避型，G3）；**方法清单必带边界声明**（G9）。全部十条全局硬闸见 spec §1.5。
